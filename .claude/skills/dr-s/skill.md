# Daily Routine Start (dr-s)

今日の習慣記録を開始します。完了した習慣があれば、引数として渡すことができます。

## 使い方

```
# テンプレート作成のみ
/dr-s

# 完了した習慣を同時に記録
/dr-s ラジオ体操 シダキュア スクワット20 腕立て20 腹筋20

# 短縮名も使える
/dr-s radio ceda squat 20 pushup 20 abs 20
```

## 実行内容

1. システム日付を確認
2. 該当日付のブランチを作成（`git checkout -b YYYY-MM-DD`）
3. `daily/YYYY-MM-DD.md` をテンプレートから作成
4. 引数があれば、該当する習慣をチェック済みにする
5. 完了メッセージを表示

## 対応する習慣名

| 正式名 | 短縮名 |
|--------|--------|
| ラジオ体操 | `ラジオ`, `radio` |
| 洗顔・保湿 | `洗顔`, `保湿`, `skincare` |
| シダキュア | `シダ`, `ceda` |
| Typnix 練習 | `typnix`, `typ` |
| スクワット | `squat`, `sq` |
| 腕立て伏せ | `腕立て`, `腕立`, `pushup`, `pu` |
| 腹筋 | `abs` |
| エクササイズ動画 | `動画`, `video`, `ex` |

## 回数の記録

筋トレ系の習慣（スクワット、腕立て伏せ、腹筋）は、回数を数字で指定できます。

```
/dr-s スクワット20 腕立て15 腹筋20
→ - [x] スクワット 20 回完了
→ - [x] 腕立て伏せ 15 回完了
→ - [x] 腹筋 20 回完了
```

## 処理の流れ

### 1. 日付とファイルの確認

```markdown
システム日付を取得（例: 2026-03-23）
daily/2026-03-23.md が存在するか確認
```

### 2. 新規作成フロー（ファイルが存在しない場合）

```markdown
1. ブランチ作成: `git checkout -b 2026-03-23`
2. テンプレート読み込み: `daily/TEMPLATE.md`
3. 日次ファイル作成: `daily/2026-03-23.md`（日付を置換）
```

### 3. 習慣のチェック

引数が渡された場合、以下のロジックでチェック済みにします：

```markdown
引数をパース:
  - スペース、カンマで分割
  - 各項目を習慣名と照合（正式名・短縮名の両方）
  - 数字があれば回数として記録

該当する習慣のチェックボックスを更新:
  - [ ] → [x]
  - 回数があれば「XX 回完了」を追記
```

### 4. 完了メッセージ

```markdown
✅ 2026-03-23 の記録を作成しました！

完了した習慣:
- ✅ ラジオ体操
- ✅ シダキュア
- ✅ スクワット 20 回完了

残りの習慣や学習内容があれば、いつでも報告してください。
```

## 注意事項

- **既にファイルが存在する場合**: ブランチ作成をスキップし、ファイルを更新モードで開く
- **引数なしの場合**: テンプレートのみ作成し、ユーザーに報告を促す
- **ブランチ作成失敗**: エラーメッセージを表示し、ユーザーに状況を報告

## 実装のポイント

1. **柔軟なパース**: カンマ、スペース、全角・半角に対応
2. **部分一致**: 「ラジオ」→「ラジオ体操」のような部分一致も許容
3. **大文字小文字**: 英語の短縮名は大文字小文字を区別しない
4. **回数の抽出**: 数字を見つけたら直前の習慣名に関連付ける
