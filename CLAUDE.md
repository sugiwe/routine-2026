# 2026 年 習慣管理プロジェクト

このディレクトリは、2026 年に毎日コツコツ取り組む習慣を管理・記録するためのものです。

## 管理する習慣

### 基本習慣

- ラジオ体操
- 洗顔・保湿
- シダキュア（花粉症の薬）
- Typnix 練習（タイピング練習アプリ）

### エクササイズ

- スクワット（目標: 20 回）
- 腕立て伏せ（目標: 20 回）
- 腹筋（目標: 20 回）
- エクササイズ動画（YouTube などの運動動画）

### 学習習慣

- **Sociomedia ヒューマンインターフェース ガイドライン**（略称: **shig**）

  - 100 トピックを 1 日 1 つずつ読んでいく
  - https://www.sociomedia.co.jp/category/shig

- **Documentation for Ruby 4.0 - Array Instance Methods**（略称: **Ruby**）
  - class Array の Instance Methods を 1 日 1 つずつ読んでいく
  - https://docs.ruby-lang.org/en/4.0/Array.html

## Claude Code の処理フロー

### ⭐ 1 日の初めの作業フロー（最優先）

ユーザーから「今日の記録を開始」「おはよう」などの声かけがあり、**その日の初めてのやり取り**の場合：

#### ステップ 1: 日付の確認

- システムの日付（`<env>` の Today's date）を確認
- 例: 2026-01-05

#### ステップ 2: 日次ファイルの存在確認

- `daily/2026-01-05.md` が存在するか確認
- **存在する場合**: そのファイルを開いて更新モードで作業
- **存在しない場合**: 次のステップへ進む（新規作成フロー）

#### ステップ 3: 新規作成フローの開始

日次ファイルが存在しない場合、以下の手順で作業を開始：

1. **ブランチの作成**

   ```bash
   git checkout -b 2026-01-05
   ```

   - ブランチ名は日付形式 `YYYY-MM-DD`
   - main ブランチから分岐

2. **日次ファイルの作成**

   - `daily/TEMPLATE.md` を読み込む
   - `daily/2026-01-05.md` として新規作成
   - 日付部分（`# YYYY-MM-DD`）を実際の日付に置換

3. **ユーザーへの確認**
   - 「2026-01-05 の記録を作成しました！」
   - 完了した習慣があれば報告を促す

#### ステップ 4: 習慣の記録

- ユーザーから報告された完了済み習慣をチェック済みにする
- 回数の記録が必要な項目（筋トレなど）は具体的な回数を記載

#### 重要な判断基準

- **「その日の初めてのやり取り」の判定**:
  - 会話の最初の方で記録開始を依頼された場合
  - まだ日次ファイルが存在しない場合
- **エラーハンドリング**:
  - ブランチ作成に失敗 → ユーザーに状況を報告
  - 日付の不一致 → システム日付を優先し、ユーザーに確認

### 2. daily/ への記録（通常の記録更新）

日次ファイルが既に存在し、追加の習慣や学習内容を記録する場合：

#### 基本習慣の更新

- ユーザーから報告された習慣のチェックボックスを `[x]` に変更
- 回数がある項目は実際の回数を記載
  - 例: `- [x] 腕立て伏せ 15 回完了`
- 目標に満たない場合でもそのまま記録（正確性を優先）

#### 学習内容の記録

1. **情報の記録**

   - トピック名/メソッド名を記載
   - URL を記載
   - ユーザーのメモを記載
   - チェックボックスを `[x]` に変更
   - **詳細ノートへのリンクを追加**（learning/ のファイルへの相対パス）

2. **次のステップへ**
   - 学習内容が記録された場合、自動的に「3. learning/ への詳細ノート作成」へ進む

### 3. learning/ への詳細ノート作成

学習内容（shig または Ruby）が記録された場合、詳細ノートを作成：

#### 前提: 連番の確認

1. **既存ファイルの確認**

   - `learning/sociomedia-hig/` または `learning/ruby-array-methods/` 内のファイルを確認
   - 最新の連番を特定（例: `004-foo.md` がある場合、次は `005`）

2. **連番の決定**
   - 新しいファイルの連番を決定（3 桁ゼロパディング）

#### shig の場合

1. **ファイル名の生成**

   - 形式: `{連番 3 桁}-{トピック名スラッグ}.md`
   - 例: `005-consistency-principle.md`
   - トピック名スラッグは英語、小文字、ハイフン区切り

2. **内容構成**

   ```markdown
   # {トピック名}

   - 日付: 2026-01-05
   - URL: https://www.sociomedia.co.jp/xxxx
   - daily 記録: [daily/2026-01-05.md](../../daily/2026-01-05.md)

   ## ユーザーの第一印象（daily より）

   {ユーザーのメモをそのまま転記}

   ## 深掘り・補足

   {Claude が追加する付加価値情報}

   - 概念の説明
   - 重要なポイント
   - 良い例と悪い例
   - 関連する原則やガイドラインなど
   ```

3. **深掘りの方針**
   - ソースページを WebFetch で取得して分析
   - UIX デザインの観点から実践的な補足を追加
   - 具体例を交えて解説

#### Ruby の場合

1. **ファイル名の生成**

   - 形式: `{連番 3 桁}-{メソッド名スラッグ}.md`
   - 例: `005-append.md`
   - メソッド名スラッグは記号を単語化（`&` → `ampersand`、`*` → `asterisk`）

2. **内容構成**

   ```markdown
   # Array#{メソッド名}

   - 日付: 2026-01-05
   - URL: https://docs.ruby-lang.org/en/4.0/Array.html#method-i-xxxx
   - daily 記録: [daily/2026-01-05.md](../../daily/2026-01-05.md)

   ## 基本情報

   {メソッドシグネチャと公式ドキュメントの概要}

   ## ユーザーの学習記録（daily より）

   ### 第一印象のメモ

   {ユーザーが daily に記録したメモをそのまま転記}

   ### コード例

   {ユーザーが daily に記録したコード例をそのまま転記}

   ## 深掘り・補足

   {Claude が追加する付加価値情報}

   - 重要なポイント
   - ユースケース
   - 注意点
   - 関連メソッドとの比較
   - パフォーマンスの考慮事項など
   ```

3. **深掘りの方針**
   - 公式ドキュメントを WebFetch で取得して詳細を分析
   - 実践的な使用例を追加
   - 落とし穴や注意点を明記

#### 日本語訳について

- 自動翻訳された日本語に明らかな誤訳がある場合は修正
- 技術用語は適切な日本語訳を使用

### 4. 連番管理の詳細

学習ノートのファイル名は 3 桁の連番（001, 002, ...）で管理します。

- 進捗が分かりやすい
- 同日に複数学習しても対応可能
- ファイルが自然にソートされる

### 5. _sidebar.md の更新（Web ページのナビゲーション）

日次記録や学習ノートを追加した場合、`_sidebar.md` を更新して Web ページのサイドバーに反映します。

#### 更新するタイミング

- 日次記録を追加した場合
- 学習ノート（shig または Ruby）を追加した場合

#### 更新方法

1. **`_sidebar.md` を開く**

2. **日次記録を追加した場合**

   ```markdown
   * **日次記録**
     * [2026-01-06](daily/2026-01-06.md)  ← 新しい日付を一番上に追加
     * [2026-01-05](daily/2026-01-05.md)
     * [2026-01-04](daily/2026-01-04.md)
     ...
   ```

   - 新しい日付を**一番上**に追加（降順）

3. **学習ノート（shig）を追加した場合**

   ```markdown
   * **学習ノート**
     * **Sociomedia HIG**
       * [006. New Topic](learning/sociomedia-hig/006-new-topic.md)  ← 新しいトピックを一番上に追加
       * [005. Natural Mapping](learning/sociomedia-hig/005-natural-mapping.md)
       ...
   ```

   - 連番とタイトルを記載
   - 新しいトピックを**一番上**に追加（降順）

4. **学習ノート（Ruby）を追加した場合**

   ```markdown
   * **学習ノート**
     * **Ruby Array Methods**
       * [006. new_method](learning/ruby-array-methods/006-new-method.md)  ← 新しいメソッドを一番上に追加
       * [005. <<](learning/ruby-array-methods/005-left-shift.md)
       ...
   ```

   - 連番とメソッド名を記載
   - 新しいメソッドを**一番上**に追加（降順）

#### 重要な注意点

- **必ず更新する**: Web ページのナビゲーションに新しい記録が表示されるため
- **降順で管理**: 新しいものが上に来るように配置
- **タイトルは簡潔に**: サイドバーで見やすいように

### 6. Git コミットと PR の作成

作業が完了したら、変更をコミットして PR を作成します。

#### コミットの作成

1. **変更内容の確認**

   ```bash
   git status
   git diff
   ```

2. **ステージングとコミット**

   ```bash
   git add daily/2026-01-05.md
   # 学習ノートも作成した場合は追加
   git add learning/sociomedia-hig/005-example.md
   git add learning/ruby-array-methods/003-example.md
   # _sidebar.md も必ず追加
   git add _sidebar.md

   git commit -m "2026-01-05 の日次記録を追加"
   ```

3. **コミットメッセージのルール**
   - **必ず日本語**で記述
   - 簡潔に変更内容を要約
   - 例:
     - `2026-01-05 の日次記録を追加`
     - `shig: 一貫性の原則の学習ノートを追加`
     - `Ruby: Array#append の学習ノートを追加`

#### PR の作成

1. **ブランチのプッシュ**

   ```bash
   git push -u origin 2026-01-05
   ```

2. **PR の作成**

   ```bash
   gh pr create --title "2026-01-05 の日次記録" --body "$(cat <<'EOF'
   ## 概要
   2026-01-05 の習慣記録と学習ノートを追加

   ## 変更内容
   - 基本習慣の記録
   - 学習内容の記録と詳細ノート作成

   🤖 Generated with [Claude Code](https://claude.com/claude-code)
   EOF
   )"
   ```

3. **PR のルール**
   - タイトル: 日本語で簡潔に（例: `2026-01-05 の日次記録`）
   - Description: 日本語で詳細を記述
     - 概要セクション
     - 変更内容セクション
     - 必要に応じてテスト計画など
   - 最後に Claude Code の署名を追加

#### 重要な注意事項

- **すべて日本語で記述**（コミットメッセージ、PR タイトル、PR description）
- 1 日の作業が完了したらすぐに PR を作成
- PR はマージせずに残しておく（レビュー・記録用）

## Web ページ化（Docsify）

このプロジェクトは Docsify を使って Web ページとして閲覧できます。

### アクセス方法

**📖 [https://sugiwe.github.io/routine-2026/](https://sugiwe.github.io/routine-2026/)**

### 特徴

- **サイドバーナビゲーション**: 日次記録と学習ノートに簡単アクセス
- **検索機能**: 過去の記録を全文検索
- **見やすい UI**: Markdown が美しく整形された状態で表示
- **コードハイライト**: Ruby のシンタックスハイライト対応
- **コピーボタン**: コードブロックをワンクリックでコピー

### 関連ファイル

- `index.html`: Docsify の設定ファイル
- `_sidebar.md`: サイドバーのナビゲーション（**要更新**）
- `.nojekyll`: GitHub Pages 用（Jekyll を無効化）

### _sidebar.md の重要性

`_sidebar.md` は Web ページのナビゲーションを定義するファイルです。**日次記録や学習ノートを追加するたびに必ず更新してください**（詳細は「5. _sidebar.md の更新」を参照）。

更新を忘れると、Web ページに新しい記録が表示されません。
