# スキル使い方ガイド

このプロジェクトには、習慣記録を効率化するための3つのスキルがあります。

## スキル一覧

| スキル | 説明 | 使用タイミング |
|--------|------|---------------|
| `/dr-s` | 記録開始 | 朝、習慣を完了したタイミング |
| `/dr-l` | 学習ノート作成 | 学習を終えた後 |
| `/dr-f` | 記録完了 | 1日の終わりにコミット・PR作成 |

---

## 1. `/dr-s` - Daily Routine Start

### 基本的な使い方

```
# テンプレートだけ作成
/dr-s

# 完了した習慣を同時に記録
/dr-s ラジオ体操 シダキュア スクワット20 腕立て20 腹筋20
```

### 対応する習慣名

| 正式名 | 短縮名 |
|--------|--------|
| ラジオ体操 | `ラジオ`, `radio` |
| 洗顔・保湿 | `洗顔`, `保湿`, `skincare` |
| シダキュア | `シダ`, `ceda` |
| Typnix 練習 | `typnix`, `typ` |
| スクワット | `squat`, `sq` |
| 腕立て伏せ | `腕立て`, `腕立`, `pushup`, `pu` |
| 腹筋 | `abs` |
| エクササイズ動画 | `動画`, `video`, `ex` |

### 回数の指定

筋トレ系は回数を続けて書けます：

```
/dr-s スクワット20 腕立て15 腹筋20
→ - [x] スクワット 20 回完了
→ - [x] 腕立て伏せ 15 回完了
→ - [x] 腹筋 20 回完了
```

### 実行される処理

1. 今日の日付を確認（例: `2026-03-23`）
2. ブランチを作成（`git checkout -b 2026-03-23`）
3. テンプレートから日次ファイルを作成（`daily/2026-03-23.md`）
4. 引数があれば該当する習慣をチェック済みにする
5. 完了メッセージを表示

---

## 2. `/dr-l` - Daily Routine Learning Note

### 基本的な使い方

```
# Sociomedia HIG の学習ノート
/dr-l shig "トピック名" https://www.sociomedia.co.jp/xxxx "メモ"

# Ruby Array Methods の学習ノート
/dr-l ruby "メソッド名" https://docs.ruby-lang.org/en/4.0/Array.html#method-i-xxx "メモ"
```

### 引数の形式

```
/dr-l <種別> <トピック名/メソッド名> <URL> [メモ]
```

- **種別**: `shig` または `ruby`
- **トピック名/メソッド名**: 学習した内容の名前
- **URL**: ソースページのURL
- **メモ**: （省略可）ユーザーの第一印象や補足

### 例

```
# Sociomedia HIG
/dr-l shig "一貫性の原則" https://www.sociomedia.co.jp/6789 "UIは統一感が大事"

# Ruby
/dr-l ruby "select" https://docs.ruby-lang.org/en/4.0/Array.html#method-i-select "配列のフィルタリング"

# メモなし
/dr-l ruby "append" https://docs.ruby-lang.org/en/4.0/Array.html#method-i-append
```

### 実行される処理

1. 今日の日次ファイルを更新（学習セクションにチェック済み・URL・メモを記録）
2. 既存の学習ノートの連番を確認
3. WebFetch でソースページを取得・分析
4. 詳細ノートを作成（`learning/` 以下）
5. 日次ファイルに詳細ノートへのリンクを追加
6. `_sidebar.md` を更新
7. 完了メッセージを表示

---

## 3. `/dr-f` - Daily Routine Finish

### 基本的な使い方

```
/dr-f
```

引数は不要です。現在のブランチと変更内容を自動的に検出し、コミット・プッシュ・PR作成を実行します。

### 実行される処理

1. 現在のブランチ名を確認（例: `2026-03-23`）
2. 変更内容を確認（`git status`, `git diff`）
3. 日次ファイルを読み込み、完了した習慣と学習内容を抽出
4. ステージング（`git add`）
5. コミット（日本語のコミットメッセージを自動生成）
6. プッシュ（`git push -u origin`）
7. PR 作成（日本語のタイトルと本文を自動生成）
8. PR URL を表示

### 生成されるコミットメッセージの例

```
2026-03-23 の日次記録を追加

## 完了した習慣
- ラジオ体操
- シダキュア
- Typnix 練習
- スクワット 20 回
- 腕立て伏せ 20 回
- 腹筋 20 回

## 学習内容
- Sociomedia HIG: 082. モバイルでは包括的より階層的に
- Ruby: Array#select

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>
```

---

## 典型的な1日の流れ

### 朝（習慣完了後）

```
/dr-s ラジオ体操 シダキュア スクワット20 腕立て20 腹筋20
```

### 学習後

```
/dr-l shig "一貫性の原則" https://www.sociomedia.co.jp/xxxx "UIは統一感が大事"
/dr-l ruby "select" https://docs.ruby-lang.org/en/4.0/Array.html#method-i-select
```

### 1日の終わり

```
/dr-f
```

これで、ブランチ作成からPR作成まで全部自動化されます！

## Tips

### 習慣を追加で記録したい場合

`/dr-s` を使った後でも、直接ファイルを編集してチェックを入れればOKです。

### 学習ノートを後から修正したい場合

`learning/` 以下のファイルを直接編集してください。`/dr-f` を実行すれば自動的にコミットされます。

### スキルの仕様を確認したい場合

`.claude/skills/` 以下の `skill.md` ファイルに詳細な仕様が書かれています。

---

## トラブルシューティング

### スキルが認識されない

Claude Code を再起動してください。スキルは起動時に読み込まれます。

### ブランチが既に存在する

`/dr-s` を再実行すると、既存のブランチとファイルを使います。

### コミットに失敗した

`/dr-f` でエラーが出た場合、手動で `git status` を確認し、必要に応じて修正してください。
