## 基本情報

```ruby
min → element
min(n) → new_array
min {|a, b| ... } → element
min(n) {|a, b| ... } → new_array
```

配列内の最小値の要素を返します。引数なしの場合は単一の要素、引数 `n` がある場合は最小の `n` 個の要素を配列で返します。

**重要**: `self` を変更しない非破壊的メソッドです。

### コード例

```ruby
# Returns one of the following:

# - The minimum-valued element from self.
# - A new array of minimum-valued elements from self.

# Does not modify self.

# With no block given, each element in self must respond to method <=> with a numeric.

# With no argument and no block, returns the element in self having the minimum value per method <=>:

[1, 0, 3, 2].min # => 0

# With non-negative numeric argument count and no block, returns a new array with at most count elements, in ascending order, per method <=>:

[1, 0, 3, 2].min(3)   # => [0, 1, 2]
[1, 0, 3, 2].min(3.0) # => [0, 1, 2]
[1, 0, 3, 2].min(9)   # => [0, 1, 2, 3]
[1, 0, 3, 2].min(0)   # => []

# With a block given, the block must return a numeric.

# With a block and no argument, calls the block self.size - 1 times to compare elements; returns the element having the minimum value per the block:

['0', '', '000', '00'].min {|a, b| a.size <=> b.size }
# => ""

# With non-negative numeric argument count and a block, returns a new array with at most count elements, in ascending order, per the block:

['0', '', '000', '00'].min(2) {|a, b| a.size <=> b.size }
# => ["", "0"]
```

## 深掘り・補足

### 重要なポイント

1. **`Array#max` の対となるメソッド**
   - ユーザーの第一印象の通り、`max` の逆の動作をします
   - 比較方法、引数、ブロックの使い方はすべて `max` と同じパターン

2. **デフォルトの比較: `<=>` 演算子**
   - ブロックなしの場合、各要素は `<=>` に応答できる必要があります
   - 数値、文字列、日付など、比較可能なオブジェクトで使えます

3. **引数 `n` の動作**
   - 最小の `n` 個を**昇順**で返します（小さい順）
   - `n` が配列のサイズより大きい場合は全要素を返します
   - `n` が 0 の場合は空配列を返します

4. **ブロックによるカスタム比較**
   - ブロックは数値を返す必要があります
   - `a <=> b` のように比較演算子を使うのが一般的
   - 文字列の長さ、オブジェクトの属性など、任意の基準で比較可能

### ユースケース

#### 基本的な最小値の取得

```ruby
# 数値配列の最小値
[5, 2, 8, 1, 9].min  # => 1

# 文字列配列の最小値（辞書順）
['cat', 'dog', 'ant', 'bee'].min  # => "ant"
```

#### 複数の最小値を取得

```ruby
# 成績の下位3人を抽出
scores = [85, 92, 78, 95, 88, 72, 90]
bottom_three = scores.min(3)  # => [72, 78, 85]

# 価格の安い順に5商品
prices = [1200, 800, 1500, 600, 2000, 900, 1100]
cheapest_five = prices.min(5)  # => [600, 800, 900, 1100, 1200]
```

#### カスタム比較の活用

```ruby
# 文字列の長さで最短のものを取得
words = ['elephant', 'cat', 'butterfly', 'dog']
shortest = words.min {|a, b| a.length <=> b.length }
# => "cat"

# ハッシュ配列を年齢で比較
people = [
  {name: 'Alice', age: 30},
  {name: 'Bob', age: 25},
  {name: 'Charlie', age: 35}
]
youngest = people.min {|a, b| a[:age] <=> b[:age] }
# => {:name=>"Bob", :age=>25}

# 年齢の若い順に2人
youngest_two = people.min(2) {|a, b| a[:age] <=> b[:age] }
# => [{:name=>"Bob", :age=>25}, {:name=>"Alice", :age=>30}]
```

### 注意点

1. **空配列の場合**
   ```ruby
   [].min  # => nil
   [].min(3)  # => []
   ```

2. **比較不可能な要素の混在**
   ```ruby
   [1, 'a', 2].min  # ArgumentError: comparison of Integer with String failed
   ```
   異なる型が混在していると比較エラーになります。

3. **ブロックは数値を返す必要がある**
   ```ruby
   # ❌ 真偽値を返すとエラー
   [1, 2, 3].min {|a, b| a < b }  # TypeError

   # ✅ 数値（-1, 0, 1）を返す
   [1, 2, 3].min {|a, b| a <=> b }  # => 1
   ```

4. **`Enumerable#min` との関係**
   - `Array` は `Enumerable` を include しているため、`Enumerable#min` も使えます
   - 挙動は同じですが、`Array#min` の方がパフォーマンスが最適化されている可能性があります

### 関連メソッドとの比較

| メソッド | 説明 | 例 |
|---------|------|-----|
| `Array#max` | 最大値を返す（対となるメソッド） | `[1, 3, 2].max` → `3` |
| `Array#minmax` | 最小値と最大値を配列で返す | `[1, 3, 2].minmax` → `[1, 3]` |
| `Array#sort` | 配列全体をソート | `[3, 1, 2].sort` → `[1, 2, 3]` |
| `Array#min_by` | ブロックの結果で最小値を返す（簡潔な記法） | `words.min_by(&:length)` |

### パフォーマンスの考慮事項

- `min` の計算量は **O(n)**（配列全体を1回走査）
- 複数の最小値を取得する `min(n)` は内部で部分ソートを行うため **O(n log k)**（k は取得する要素数）
- 単に最小値1つを取得する場合は `min` が最も効率的
- 配列全体のソートが必要な場合は `sort` を使う方が適切

### `Array#max` との対称性

昨日学習した `Array#max` と完全に対称的な動作をします：

| 観点 | `Array#max` | `Array#min` |
|------|-------------|-------------|
| 引数なし | 最大値1つ | 最小値1つ |
| 引数 `n` | 最大の `n` 個（降順） | 最小の `n` 個（昇順） |
| ブロック | カスタム比較で最大値 | カスタム比較で最小値 |
| 戻り値の順序 | 大→小 | 小→大 |

この対称性により、`max` と `min` を組み合わせて使うコードも書きやすくなっています。
