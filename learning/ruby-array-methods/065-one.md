## 基本情報

```ruby
one? → true or false
one? {|element| ... } → true or false
one?(object) → true or false
```

**配列の要素のうち、指定された条件を満たすものがちょうど1つ存在する場合に `true` を返します。**

### コード例

```ruby
# Returns true if exactly one element of self meets a given criterion.
# 自身の要素のうち、指定された条件を満たすものがちょうど1つ存在する場合にtrueを返します。

# With no block given and no argument, returns true if self has exactly one truthy element, false otherwise:
# ブロックも引数も指定しない場合、自身がただ1つの真値要素を持つなら真を返し、それ以外なら偽を返します。
[nil, 0].one? # => true
[0, 0].one? # => false
[nil, nil].one? # => false
[].one? # => false

# With a block given, calls the block with each element in self; returns true if the block a truthy value for exactly one element, false otherwise:
# ブロックが与えられた場合、自身の各要素を引数としてブロックを呼び出します。ブロックが真の値を返した要素がちょうど1つだけであれば true を返し、それ以外の場合は false を返します。
[0, 1, 2].one? {|element| element > 0 } # => false
[0, 1, 2].one? {|element| element > 1 } # => true
[0, 1, 2].one? {|element| element > 2 } # => false

# With argument object given, returns true if for exactly one element element, object === element; false otherwise:
# 引数`object`が与えられた場合、要素`element`のうちただ1つだけが`object === element`を満たしていれば`true`を返し、それ以外の場合は`false`を返します。
[0, 1, 2].one?(0) # => true
[0, 0, 1].one?(0) # => false
[1, 1, 2].one?(0) # => false
['food', 'drink'].one?(/bar/) # => false
['food', 'drink'].one?(/foo/) # => true
[].one?(/foo/) # => false
```

## 深掘り・補足

### 重要なポイント

1. **「ちょうど1つ」の判定**
   - 0個でも2個以上でもダメ、**exactly one** が条件
   - 条件を満たす要素が複数あると `false` を返す

2. **3つの使い方**
   - 引数なし・ブロックなし：真値要素が1つだけか判定
   - ブロック付き：ブロックが真を返す要素が1つだけか判定
   - 引数付き：`===` で一致する要素が1つだけか判定

3. **真値の判定**
   - Ruby では `nil` と `false` のみが偽
   - `0` や空文字列 `""` は真値として扱われる

### 実践的なユースケース

#### 1. バリデーション：ユニークな要素の確認

```ruby
# エラーメッセージのリストで、特定のエラーが1つだけ発生しているか確認
errors = ['timeout', 'retry', 'timeout']
errors.one?('timeout') # => false（2つある）

errors = ['timeout', 'retry', 'success']
errors.one?('timeout') # => true（1つだけ）
```

#### 2. フラグの排他制御

```ruby
# 複数のフラグのうち、ちょうど1つだけが有効か確認
flags = [false, true, false]
flags.one? # => true（1つだけ真）

flags = [false, true, true]
flags.one? # => false（2つ真）
```

#### 3. データの整合性チェック

```ruby
# ユーザーのロールが1つだけ設定されているか確認
user_roles = ['admin', 'editor', 'viewer']
user_roles.one? {|role| role == 'admin' } # => true

# 重複チェック
user_roles = ['admin', 'admin', 'viewer']
user_roles.one? {|role| role == 'admin' } # => false
```

### 関連メソッドとの比較

| メソッド | 条件 | 返り値 |
|---------|------|--------|
| `all?` | **すべての**要素が条件を満たす | `true` / `false` |
| `any?` | **少なくとも1つ**の要素が条件を満たす | `true` / `false` |
| `none?` | **どの要素も**条件を満たさない | `true` / `false` |
| `one?` | **ちょうど1つだけ**の要素が条件を満たす | `true` / `false` |

```ruby
arr = [1, 2, 3]

arr.all? {|n| n > 0 }  # => true（すべて > 0）
arr.any? {|n| n > 2 }  # => true（3が該当）
arr.none? {|n| n > 5 } # => true（どれも該当しない）
arr.one? {|n| n > 2 }  # => true（3だけが該当）
```

### 注意点

1. **空配列の扱い**
   ```ruby
   [].one? # => false（1つもないので false）
   ```

2. **ブロックなしの場合は真値判定**
   ```ruby
   [nil, 0].one? # => true（0は真値なので、真値が1つ）
   [nil, false].one? # => false（真値が0個）
   ```

3. **`===` の挙動**
   ```ruby
   # 正規表現を使うと === が活用される
   ['foo', 'bar'].one?(/foo/) # => true
   ['foo', 'foobar'].one?(/foo/) # => false（2つマッチ）
   ```

### パフォーマンスの考慮

`one?` メソッドは**早期リターン（early return）を実装**しており、効率的に動作します：

1. **2つ目が見つかった時点で即座に `false` を返す**
   - 条件に一致する要素が2つ見つかると、残りの要素を評価せずに停止
   - 大きな配列でも、条件を満たす要素が前方にあればすぐに結果が出る

2. **配列全体を走査するのは以下の場合のみ**
   - 条件に一致する要素がちょうど1つの場合（結果が `true`）
   - 条件に一致する要素が0個の場合（結果が `false`）

```ruby
# 早期リターンの効果
count = 0
(1..1_000_000).to_a.one? do |n|
  count += 1
  n > 10  # 11, 12, 13... と多数該当
end
# => false
# count は 12 のみ！（11と12を見つけた時点で停止）

# 全走査が必要なケース
count = 0
(1..1_000_000).to_a.one? { |n| count += 1; n == 500_000 }
# => true
# count は 1_000_000（最後まで確認が必要）
```

この早期リターン動作により、多数の要素が条件を満たす配列に対しても高速に処理できます。
