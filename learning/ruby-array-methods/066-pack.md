## 基本情報

```ruby
pack(template) -> String
pack(template, buffer: buf) -> String
```

配列の各要素をテンプレート文字列の指定に従ってバイナリ形式にパックし、バイナリ文字列として返します。

### メソッドシグネチャ

- **引数**: `template`（必須）- フォーマット指示のテンプレート文字列
- **引数**: `buffer`（オプション）- 結果を格納する String オブジェクト（メモリ再利用）
- **戻り値**: バイナリ文字列（String）

### 公式ドキュメントの概要（英語）

Formats each element in self into a binary string; returns that string.

### 日本語の説明

配列の内容をテンプレート文字列に従ってバイナリとしてパックした文字列を返します。テンプレート文字列はディレクティブ（指示文字）の並びで構成されます。

## 深掘り・補足

### 用途と使いどころ

`pack` は主に以下のような場面で使用されます：

1. **バイナリデータの生成**
   - ネットワークパケットの構築
   - バイナリファイルの作成
   - 低レベルなデータ構造の操作

2. **エンコーディング変換**
   - Base64 エンコード
   - UUエンコード
   - バイトデータから文字列への変換

3. **データシリアライゼーション**
   - データの送信前の準備
   - ファイルへの保存

### 主要なテンプレート文字

#### 文字列型

```ruby
["abc"].pack("a4")    # => "abc\x00"  (ヌル文字で充填)
["abc"].pack("A4")    # => "abc "     (スペースで充填)
["abc"].pack("Z4")    # => "abc\x00"  (ヌル終端文字列)
```

#### 整数型

```ruby
[1, 2, 3].pack("c*")   # => "\x01\x02\x03"  (8ビット符号付き整数)
[1, 2].pack("n2")      # => "\x00\x01\x00\x02"  (16ビット ビッグエンディアン)
[1, 2].pack("v2")      # => "\x01\x00\x02\x00"  (16ビット リトルエンディアン)
```

#### 特殊な例：文字列からバイト配列へ

```ruby
[82, 117, 98, 121].pack("c*")  # => "Ruby"
# ASCII コードの配列から文字列を生成
```

#### エンコーディング

```ruby
["Hello"].pack("m")    # => "SGVsbG8=\n"  (Base64 エンコード)
["Hello"].pack("u")    # => "%2&5L;&\\n"  (UUエンコード)
```

### テンプレート文字の修飾子

- `*`: 残りのすべての要素に適用
- `数値`: 指定回数繰り返し
- `_`: ネイティブサイズを使用（例: `s_` でネイティブ short）
- `!`: ネイティブサイズを使用（`_` と同じ）

```ruby
[1, 2, 3, 4].pack("c*")   # すべての要素
[1, 2, 3, 4].pack("c2")   # 最初の2要素のみ
```

### 重要なポイント

1. **テンプレート内の空白は無視される**
   ```ruby
   [1, 2].pack("n n")  # "n n" は "nn" と同じ
   ```

2. **コメントが使える**
   ```ruby
   [1, 2].pack("n  # first\nn  # second")
   ```

3. **逆変換は String#unpack**
   ```ruby
   str = [1, 2, 3].pack("c*")
   str.unpack("c*")  # => [1, 2, 3]
   ```

4. **buffer オプションでメモリ効率化**
   ```ruby
   buf = String.new
   [1, 2].pack("c*", buffer: buf)
   # buf が再利用される（新しい String を作らない）
   ```

### よくある使用例

#### 1. バイナリファイルの書き込み

```ruby
data = [0xFF, 0xD8, 0xFF, 0xE0]  # JPEG のマジックナンバー
File.binwrite("test.jpg", data.pack("C*"))
```

#### 2. ネットワークバイトオーダー

```ruby
# IPアドレスをバイナリ形式に変換
ip_parts = [192, 168, 1, 1]
binary_ip = ip_parts.pack("C4")
```

#### 3. Base64 エンコード

```ruby
["Hello, World!"].pack("m")
# => "SGVsbG8sIFdvcmxkIQ==\n"
```

### パフォーマンスの考慮事項

- `buffer` オプションを使うとメモリアロケーションを削減できる
- 大量のデータを扱う場合は、テンプレート文字の選択が重要
- `*` 修飾子は便利だが、配列のサイズによってはパフォーマンスに影響

### 関連メソッド

- **String#unpack**: `pack` の逆操作（バイナリ文字列を配列に戻す）
- **String#unpack1**: 最初の要素のみ取り出す
- **Array#join**: 単純な文字列結合（バイナリ変換なし）

### 注意点

- テンプレート文字列の指定を間違えると、予期しないバイナリデータになる
- エンディアンの違いに注意（特にネットワーク通信時）
- プラットフォーム依存のテンプレート文字（`i`, `I`, `l`, `L` など）は移植性に影響
