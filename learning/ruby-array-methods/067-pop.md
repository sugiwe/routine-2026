## 基本情報

```ruby
pop → object or nil
pop(n) → new_array
```

配列の末尾の要素を削除して返すメソッドです。引数なしの場合は単一の要素を、引数ありの場合は指定した個数の要素を含む配列を返します。

### 第一印象のメモ

### コード例

```ruby
# Removes and returns trailing elements of self.
# 自身の末尾の要素を削除して返します。
# With no argument given, removes and returns the last element, if available; otherwise returns nil:
# 引数を与えなかった場合、末尾の要素があれば削除して返します。なければnilを返します。
a = [:foo, 'bar', 2]
a.pop  # => 2
a      # => [:foo, "bar"]
[].pop # => nil

# With non-negative integer argument count given, returns a new array containing the trailing count elements of self, as available:
# 非負の整数である引数countが与えられた場合、自身の末尾からcount個の要素を含む新しい配列を、利用可能な範囲で返します。
a = [:foo, 'bar', 2]
a.pop(2) # => ["bar", 2]
a        # => [:foo]

a = [:foo, 'bar', 2]
a.pop(50) # => [:foo, "bar", 2]
a         # => []

# Related: Array#push; see also Methods for Deleting.
```

## 深掘り・補足

### 重要なポイント

**破壊的メソッド**

`pop` は元の配列を変更する破壊的メソッドです。非破壊的な方法で末尾要素を取得したい場合は `array[-1]` や `array.last` を使用してください。

**2 つの動作モード**

1. **引数なし**: 末尾の 1 要素を削除して返す（戻り値は要素そのもの）
2. **引数あり**: 末尾から指定個数を削除して配列で返す（戻り値は配列）

**配列のサイズを超えた指定**

引数に配列のサイズを超える数を指定しても、エラーにならずに全要素を返します：

```ruby
a = [1, 2, 3]
a.pop(100)  # => [1, 2, 3]
a           # => []
```

### ユースケース

**スタック構造の実装**

`pop` と `push` を組み合わせることで、LIFO（Last In, First Out）のスタックを実装できます：

```ruby
stack = []
stack.push(1)
stack.push(2)
stack.push(3)
stack.pop  # => 3
stack.pop  # => 2
```

**配列の段階的処理**

配列を末尾から順に処理していく場合に便利です：

```ruby
tasks = ["task1", "task2", "task3"]
while task = tasks.pop
  puts "Processing: #{task}"
end
# Processing: task3
# Processing: task2
# Processing: task1
```

**最後の N 個を取得して削除**

配列の末尾 N 個を取得しつつ、元の配列からは削除したい場合：

```ruby
numbers = [1, 2, 3, 4, 5]
last_two = numbers.pop(2)  # => [4, 5]
numbers                     # => [1, 2, 3]
```

### 注意点

**空配列での動作**

空配列に対して `pop` を実行すると `nil` を返します（引数なしの場合）：

```ruby
[].pop     # => nil
[].pop(2)  # => []
```

**戻り値の型の違い**

引数の有無で戻り値の型が変わることに注意：

```ruby
a = [1, 2, 3]
a.pop      # => 3 (Integer)
a.pop(1)   # => [2] (Array)
```

### 関連メソッドとの比較

| メソッド | 操作位置 | 破壊的 | 用途 |
|---------|---------|--------|------|
| `pop` | 末尾から削除 | ○ | スタック（LIFO） |
| `push` | 末尾に追加 | ○ | スタック（LIFO） |
| `shift` | 先頭から削除 | ○ | キュー（FIFO） |
| `unshift` | 先頭に追加 | ○ | キュー（FIFO） |
| `last` | 末尾を参照 | × | 削除せずに確認 |

### パフォーマンスの考慮事項

**高速な操作**

配列の末尾からの削除は、Ruby の内部実装では非常に高速です（O(1)）。先頭からの削除（`shift`）と比較すると、`pop` は配列の再配置が不要なため効率的です。

**メモリ効率**

大量の要素を `pop` で処理する場合、配列のメモリは即座には解放されない可能性があります。メモリを意識する場合は、適宜 GC の動作を考慮してください。
