# Array#sum

## 基本情報

```ruby
sum(init = 0) → number
sum(init = 0) {|element| ... } → number
```

配列の要素を合計するメソッドです。`init` 値と配列の全要素を加算した結果を返します。

## ユーザーの学習記録（daily より）

### コード例

```ruby
# With no block given, returns the sum of init and all elements of self; for array array and value init, equivalent to:
# ブロックが与えられない場合、initとselfのすべての要素の合計を返します。配列arrayと値initの場合、これは以下と同等です：
sum = init
array.each {|element| sum += element }
sum

# For example, `[e0, e1, e2].sum` returns `init + e0 + e1 + e2`.`

# Examples:

[0, 1, 2, 3].sum                 # => 6
[0, 1, 2, 3].sum(100)            # => 106
['abc', 'def', 'ghi'].sum('jkl') # => "jklabcdefghi"
[[:foo, :bar], ['foo', 'bar']].sum([2, 3])
# => [2, 3, :foo, :bar, "foo", "bar"]

# The `init` value and elements need not be numeric, but must all be `+`-compatible:
# `init`の値や各要素は数値である必要はありませんが、すべて「`+`」演算に対応している必要があります。
# Raises TypeError: Array can't be coerced into Integer.
# TypeError: ArrayをIntegerに変換できません（というエラーが発生します）。
[[:foo, :bar], ['foo', 'bar']].sum(2)

# With a block given, calls the block with each element of `self`; the block's return value (instead of the element itself) is used as the addend:
# ブロックが与えられた場合、`self` の各要素に対してそのブロックを呼び出します。その際、要素そのものではなく、ブロックの戻り値が加数として使用されます。
['zero', 1, :two].sum('Coerced and concatenated: ') {|element| element.to_s }
# => "Coerced and concatenated: zero1two"

# Notes:
# - Array#join and Array#flatten may be faster than Array#sum for an array of strings or an array of arrays.
# - Array#sum method may not respect method redefinition of "+" methods such as Integer#+.
```

## 深掘り・補足

### 重要なポイント

#### 1. `init` パラメータの役割

`init` は合計の初期値です。デフォルトは `0` ですが、任意の値を指定できます：

```ruby
[1, 2, 3].sum     # => 6 (0 + 1 + 2 + 3)
[1, 2, 3].sum(10) # => 16 (10 + 1 + 2 + 3)
```

#### 2. 数値以外にも使える

`+` 演算子に対応していれば、数値以外のデータ型でも使用できます：

```ruby
# 文字列の連結
['Hello', ' ', 'World'].sum('')  # => "Hello World"

# 配列の連結
[[1, 2], [3, 4]].sum([])  # => [1, 2, 3, 4]
```

#### 3. ブロックを使った変換

ブロックを渡すと、各要素を変換してから加算できます：

```ruby
# 長さの合計
['a', 'bb', 'ccc'].sum(0) {|s| s.length }  # => 6

# 文字列化してから連結
[1, 2, 3].sum('') {|n| n.to_s }  # => "123"
```

### ユースケース

#### 数値の合計を求める

```ruby
prices = [100, 200, 300]
total = prices.sum  # => 600
```

#### 条件付きの合計

```ruby
# 偶数のみ合計
[1, 2, 3, 4, 5].sum(0) {|n| n.even? ? n : 0 }  # => 6

# または select と組み合わせて
[1, 2, 3, 4, 5].select(&:even?).sum  # => 6
```

#### 属性値の合計

```ruby
products = [
  {name: 'A', price: 100},
  {name: 'B', price: 200},
  {name: 'C', price: 300}
]

total_price = products.sum(0) {|p| p[:price] }  # => 600
```

### 注意点

#### 1. 型の互換性

`init` と各要素は `+` 演算で互換性がある必要があります：

```ruby
# エラー例
[[:a], [:b]].sum(0)  # TypeError: Array can't be coerced into Integer
```

正しくは：

```ruby
[[:a], [:b]].sum([])  # => [:a, :b]
```

#### 2. パフォーマンスの考慮

ドキュメントの注意点にあるように：

- **文字列の配列**：`Array#join` の方が高速
- **配列の配列**：`Array#flatten` の方が高速

```ruby
# 文字列の連結
strings = ['a', 'b', 'c']
strings.sum('')   # 動作するが遅い
strings.join      # より高速

# 配列の平坦化
arrays = [[1, 2], [3, 4]]
arrays.sum([])    # 動作するが遅い
arrays.flatten    # より高速
```

#### 3. メソッドの再定義

`Integer#+` などのメソッドを再定義しても、`Array#sum` がそれを尊重しない可能性があります。

### 関連メソッドとの比較

| メソッド | 用途 | パフォーマンス |
|---------|------|--------------|
| `sum` | 汎用的な合計 | 通常 |
| `join` | 文字列の連結 | 高速 |
| `flatten` | 配列の平坦化 | 高速 |
| `reduce(:+)` | カスタム集約 | 柔軟だが遅い |

### まとめ

`Array#sum` は配列の要素を合計するシンプルで強力なメソッドです。数値だけでなく、文字列や配列など `+` 演算子に対応する型であれば何でも使えます。ただし、文字列や配列の場合は専用のメソッド（`join`, `flatten`）の方がパフォーマンスが良いため、用途に応じて使い分けることが重要です。
