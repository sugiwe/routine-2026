## 基本情報

```ruby
to_a → self or new_array
```

`Array#to_a` は、配列を配列に変換するメソッドです。一見すると冗長に見えますが、実際には重要な用途があります。

### コード例

```ruby
# When self is an instance of Array, returns self.
# self が Array のインスタンスである場合、self を返します。
# Otherwise, returns a new array containing the elements of self:
# それ以外の場合は、self の要素を含む新しい配列を返します。
class MyArray < Array; end
my_a = MyArray.new(['foo', 'bar', 'two'])
a = my_a.to_a
a # => ["foo", "bar", "two"]
a.class # => Array # Not MyArray.

# Related: see Methods for Converting.
```

## 深掘り・補足

### 重要なポイント

#### 1. Array のサブクラスを純粋な Array に変換

`Array#to_a` の主な用途は、**Array のサブクラスを純粋な Array インスタンスに変換する**ことです。

```ruby
class MyArray < Array; end
my_a = MyArray.new(['foo', 'bar', 'two'])

my_a.class        # => MyArray
my_a.to_a.class   # => Array  # 純粋な Array に変換される
```

#### 2. Array インスタンスには self を返す

通常の Array インスタンスに対しては、`to_a` は `self` を返すだけです（新しい配列を作成しない）。

```ruby
a = [1, 2, 3]
b = a.to_a

a.object_id == b.object_id  # => true（同じオブジェクト）
```

### ユースケース

#### 1. サブクラスの正規化

ライブラリやメソッドが「純粋な Array インスタンス」を期待している場合、`to_a` で変換できます。

```ruby
class SpecialArray < Array
  def special_method
    "I'm special!"
  end
end

special = SpecialArray.new([1, 2, 3])
normal = special.to_a

normal.class  # => Array
# normal.special_method  # => NoMethodError（メソッドが存在しない）
```

#### 2. ダックタイピングと互換性

Ruby の `to_a` メソッドは、配列的なオブジェクトを配列に変換するための共通インターフェースです。多くのクラスが `to_a` を実装しています：

```ruby
(1..5).to_a           # => [1, 2, 3, 4, 5]
{a: 1, b: 2}.to_a     # => [[:a, 1], [:b, 2]]
"hello".chars.to_a    # => ["h", "e", "l", "l", "o"]
```

### 注意点

#### 1. 浅いコピーではない（Array の場合）

通常の Array に対しては `self` を返すため、新しい配列は作成されません。コピーが必要な場合は `dup` や `clone` を使います。

```ruby
a = [1, 2, 3]
b = a.to_a
a << 4
b  # => [1, 2, 3, 4]（同じオブジェクトなので影響を受ける）

c = a.dup
a << 5
c  # => [1, 2, 3, 4]（別オブジェクトなので影響を受けない）
```

#### 2. 要素自体はコピーされない

サブクラスから変換する場合でも、要素自体は新しいオブジェクトになりません（要素への参照がコピーされる）。

```ruby
class MyArray < Array; end
my_a = MyArray.new([{x: 1}, {y: 2}])
a = my_a.to_a

my_a[0][:z] = 3
a[0]  # => {:x=>1, :z=>3}（要素は共有されている）
```

### 関連メソッド

#### `to_ary` との違い

- **`to_a`**: 明示的な変換メソッド（ユーザーが呼び出す）
- **`to_ary`**: 暗黙的な変換メソッド（Ruby が自動的に呼び出す）

```ruby
class ArrayLike
  def to_a
    [1, 2, 3]
  end
end

obj = ArrayLike.new
obj.to_a  # => [1, 2, 3]

# to_ary がないと、配列展開時にエラーになる
# a, b, c = obj  # => TypeError: no implicit conversion
```

#### その他の変換メソッド

- **`Array(obj)`**: オブジェクトを配列に変換（`to_a` または `to_ary` を呼び出す）
- **`[*obj]`**: スプラット演算子による配列化（`to_a` を呼び出す）

### パフォーマンスの考慮事項

- **Array インスタンス**: `self` を返すだけなので、オーバーヘッドはほぼゼロ
- **サブクラス**: 新しい配列を作成するため、O(n) の時間とメモリが必要

大きな配列のサブクラスを扱う場合は、不必要な `to_a` 呼び出しを避けるとパフォーマンスが向上します。
