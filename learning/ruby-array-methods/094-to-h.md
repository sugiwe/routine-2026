# Array#to_h

- 日付: 2026-04-04

## 基本情報

```ruby
to_h → hash
to_h {|element| ... } → hash
```

`Array#to_h` は、配列を元に新しいハッシュを生成するメソッドです。配列の要素をキーと値のペアに変換してハッシュを作成します。

## ユーザーの学習記録（daily より）

### コード例

```ruby
# Returns a new hash formed from self.
# 自身から生成された新しいハッシュを返します。
# With no block given, each element of self must be a 2-element sub-array; forms each sub-array into a key-value pair in the new hash:
# ブロックが指定されていない場合、self の各要素は2つの要素を持つサブ配列である必要があります。各サブ配列を新しいハッシュのキーと値のペアとして生成します。
a = [['foo', 'zero'], ['bar', 'one'], ['baz', 'two']]
a.to_h # => {"foo"=>"zero", "bar"=>"one", "baz"=>"two"}
[].to_h # => {}

# With a block given, the block must return a 2-element array; calls the block with each element of self; forms each returned array into a key-value pair in the returned hash:
# ブロックが与えられた場合、そのブロックは2要素の配列を返す必要があります。自身（レシーバ）の各要素に対してブロックを呼び出し、返された各配列をキーと値のペアとしてハッシュを生成します。
a = ['foo', :bar, 1, [2, 3], {baz: 4}]
a.to_h {|element| [element, element.class] }
# => {"foo"=>String, :bar=>Symbol, 1=>Integer, [2, 3]=>Array, {:baz=>4}=>Hash}

# Related: see Methods for Converting.
```

## 深掘り・補足

### ブロックなしの動作

ブロックを指定しない場合、配列の各要素は **2 要素のサブ配列** である必要があります。各サブ配列の最初の要素がキー、2 番目の要素が値になります。

```ruby
# 2要素の配列をハッシュに変換
pairs = [[:name, 'Alice'], [:age, 30], [:city, 'Tokyo']]
pairs.to_h
# => {:name=>"Alice", :age=>30, :city=>"Tokyo"}

# 空の配列は空のハッシュになる
[].to_h # => {}
```

**注意点**：要素が2要素の配列でない場合はエラーになります。

```ruby
# エラーになる例
[[1, 2, 3]].to_h  # => TypeError (wrong array length at 0)
[[1]].to_h        # => ArgumentError (wrong array length at 0)
['foo'].to_h      # => TypeError (can't convert String to Array)
```

### ブロックありの動作

ブロックを指定すると、各要素に対してブロックを実行し、その返り値（2 要素の配列）をキーと値のペアとして使用します。

```ruby
# 各要素をキーに、そのクラスを値にする
items = ['hello', 42, :symbol, [1, 2]]
items.to_h {|item| [item, item.class] }
# => {"hello"=>String, 42=>Integer, :symbol=>Symbol, [1, 2]=>Array}

# インデックスをキー、要素を値にする
fruits = ['apple', 'banana', 'cherry']
fruits.map.with_index { |fruit, i| [i, fruit] }.to_h
# => {0=>"apple", 1=>"banana", 2=>"cherry"}
```

### 実践的なユースケース

#### 1. クエリパラメータの構築

```ruby
params = [[:q, 'ruby'], [:page, 2], [:per_page, 20]]
params.to_h
# => {:q=>"ruby", :page=>2, :per_page=>20}
```

#### 2. 設定の変換

```ruby
config_array = [['host', 'localhost'], ['port', 3000], ['debug', true]]
config = config_array.to_h
# => {"host"=>"localhost", "port"=>3000, "debug"=>true}
```

#### 3. グループ化した結果の集約

```ruby
# 各要素を最初の文字でグループ化し、カウントする
words = ['apple', 'apricot', 'banana', 'blueberry', 'cherry']
words.group_by {|w| w[0] }.to_h {|letter, group| [letter, group.size] }
# => {"a"=>2, "b"=>2, "c"=>1}
```

#### 4. オブジェクトから属性を抽出

```ruby
users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },
  { id: 3, name: 'Charlie' }
]

# id をキー、name を値にするハッシュを作成
users.to_h {|user| [user[:id], user[:name]] }
# => {1=>"Alice", 2=>"Bob", 3=>"Charlie"}
```

### 関連メソッドとの比較

#### `Hash[]` との比較

```ruby
pairs = [[:a, 1], [:b, 2]]

# どちらも同じ結果
pairs.to_h        # => {:a=>1, :b=>2}
Hash[pairs]       # => {:a=>1, :b=>2}

# Hash[] はフラット配列も受け取れる
Hash[:a, 1, :b, 2]  # => {:a=>1, :b=>2}
```

#### `each_with_object` との比較

```ruby
items = ['a', 'b', 'c']

# to_h を使う場合（シンプル）
items.each_with_index.to_h
# => {"a"=>0, "b"=>1, "c"=>2}

# each_with_object を使う場合（より柔軟）
items.each_with_object({}).with_index {|(item, hash), i| hash[item] = i }
# => {"a"=>0, "b"=>1, "c"=>2}
```

`to_h` はブロックが 2 要素の配列を返す必要がありますが、`each_with_object` は自由にハッシュを構築できるため、複雑な条件分岐がある場合は後者の方が便利です。

### パフォーマンスの考慮事項

- `to_h` は新しいハッシュを作成するため、元の配列は変更されません（非破壊的）
- 大きな配列を変換する場合、メモリ使用量に注意が必要
- ブロックを使用する場合、各要素に対してブロックが実行されるため、計算量は O(n)

### 注意点とエッジケース

#### 重複キーの扱い

```ruby
# 後から来たペアが優先される
pairs = [[:a, 1], [:b, 2], [:a, 3]]
pairs.to_h
# => {:a=>3, :b=>2}  # :a の値は 3 になる
```

#### nil の扱い

```ruby
# nil もキーや値として使える
[[nil, 'value'], ['key', nil]].to_h
# => {nil=>"value", "key"=>nil}
```

#### ブロックの返り値チェック

```ruby
# ブロックが 2 要素の配列を返さない場合はエラー
[1, 2, 3].to_h {|x| x }  # => TypeError (wrong element type Integer)
[1, 2, 3].to_h {|x| [x] }  # => ArgumentError (wrong array length at 0)
```

### まとめ

`Array#to_h` は、配列をハッシュに変換する便利なメソッドです。特にブロック付きで使うと、柔軟な変換が可能になります。

**使いどころ**：
- ✅ 2 要素のサブ配列をハッシュに変換
- ✅ 配列の各要素から動的にキーと値を生成
- ✅ データ変換や集約処理

**注意点**：
- ❌ ブロックなしの場合、要素が 2 要素の配列でないとエラー
- ❌ ブロックありの場合、ブロックが 2 要素の配列を返さないとエラー
- ⚠️ 重複キーがある場合、後から来た値が優先される
