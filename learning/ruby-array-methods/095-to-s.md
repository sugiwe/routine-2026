# Array#to_s

## 基本情報

```ruby
to_s → new_string
```

配列の各要素に対してメソッド `inspect` を呼び出し、生成された新しい文字列を返します。

**注意**: `Array#to_s` は `Array#inspect` のエイリアスです。

# Returns the new string formed by calling method inspect on each array element:

a = [:foo, 'bar', 2]

## 深掘り・補足

### ユーザーの疑問に答える

確かに「inspect を呼び出す」という説明だけでは何をするメソッドなのかわかりにくいですね 😅

**`to_s` が実際にやっていること**:
- 配列を人間が読みやすい文字列表現に変換する
- デバッグやログ出力で配列の内容を確認するときに便利
- **配列全体を文字列化する**（各要素を文字列に変換して連結するわけではない）

### `inspect` との関係

```ruby
a = [:foo, 'bar', 2]
a.to_s      # => "[:foo, \"bar\", 2]"
a.inspect   # => "[:foo, \"bar\", 2]"

# 完全に同じ結果！なぜなら to_s は inspect のエイリアスだから
```

### 他の to_s メソッドとの違い

```ruby
# String#to_s は自分自身を返す
"hello".to_s # => "hello"

# Integer#to_s は数値を文字列に変換
42.to_s # => "42"

# Array#to_s は配列全体を inspect 形式で文字列化
[1, 2, 3].to_s # => "[1, 2, 3]"
```

### join との違い

```ruby
a = ['foo', 'bar', 'baz']

# to_s は配列全体を inspect 形式で表現
a.to_s # => "[\"foo\", \"bar\", \"baz\"]"

# join は各要素を連結して1つの文字列にする
a.join # => "foobarbaz"
a.join(', ') # => "foo, bar, baz"
```

**用途の違い**:
- `to_s`: デバッグ・ログ出力など、配列の構造を視覚的に確認したいとき
- `join`: 配列の要素を連結して1つの文字列を作りたいとき（CSV生成、リスト表示など）

### 実用的なユースケース

#### 1. デバッグ出力

```ruby
numbers = [1, 2, 3, 4, 5]
puts "Numbers: #{numbers.to_s}"
# => Numbers: [1, 2, 3, 4, 5]
```

#### 2. ログ記録

```ruby
users = [{ name: 'Alice' }, { name: 'Bob' }]
logger.info("Users: #{users.to_s}")
# => Users: [{:name=>"Alice"}, {:name=>"Bob"}]
```

#### 3. 文字列補間での暗黙的な呼び出し

```ruby
a = [1, 2, 3]
"Array: #{a}"  # to_s が暗黙的に呼ばれる
# => "Array: [1, 2, 3]"
```

### 注意点

- **to_s はデバッグ用**: プロダクションコードでユーザーに見せる文字列を作るには `join` や他のメソッドを使う
- **to_s は inspect のエイリアス**: 完全に同じ動作をするので、どちらを使っても OK（`inspect` のほうが意図が明確かも）
- **パフォーマンス**: 大きな配列を to_s で文字列化すると重くなる可能性がある

### まとめ

`Array#to_s` は「配列を人間が読みやすい文字列に変換する」メソッドで、`inspect` のエイリアスです。確かに「inspect を呼び出す」という説明だけでは不親切ですが、実際には **デバッグやログ出力で配列の内容を視覚的に確認するための便利なメソッド** です。ユーザーに見せる文字列を作る場合は `join` を使いましょう！
