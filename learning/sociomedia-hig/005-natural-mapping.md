# 5. マッピング (Natural Mapping)

- 日付: 2026-01-05
- URL: https://www.sociomedia.co.jp/9049

## メモ（daily より）

操作するところと結果が反映されるところの対応関係が把握できるような手がかり（位置関係・形・色・記号）を与える

## 深掘り・補足

### 概念の説明

**Natural Mapping（自然なマッピング）** とは、操作部と結果表示部の対応関係を、ユーザーが直感的に理解できるようにするデザイン原則です。物理的な配置、視覚的な手がかり、記号などを活用して、「どの操作が何に影響するか」を明確にします。

### 重要なポイント

1. **直感的な対応関係**: ユーザーが考えなくても、操作と結果の関係が理解できる
2. **視覚的手がかりの活用**: 位置関係、形、色、記号などを効果的に使う
3. **認知負荷の軽減**: 試行錯誤や記憶に頼らず、見ただけで理解できる

### 良い例と悪い例

#### 良い例: ガスコンロ
- スイッチとバーナーの配置が対応している
- 左のスイッチ → 左のバーナー、右のスイッチ → 右のバーナー
- 物理的な位置関係が自然なマッピングを実現

#### 悪い例: 配置が不明瞭なコンロ
- 4つのバーナーに対して、スイッチが一直線に並んでいる
- どのスイッチがどのバーナーを制御するか、ラベルを読まないとわからない
- ユーザーは毎回確認が必要で、間違いやすい

### UI デザインでの応用

1. **フォームのレイアウト**
   - ラベルと入力欄を近接配置
   - エラーメッセージは該当フィールドの近くに表示

2. **ボリュームコントロール**
   - 左右のスピーカーアイコンと対応するスライダーを同じ位置に配置
   - 上下スライダー: 上が音量大、下が音量小

3. **タブとコンテンツ**
   - 選択されたタブが、表示されているコンテンツと明確に対応
   - 視覚的な接続（色、位置、線など）でマッピングを強化

### 関連する原則

- **シグニファイア**: マッピングを示す視覚的手がかり
- **メンタルモデル**: ユーザーの期待に沿った配置
- **視覚ゲシュタルト**: 近接性の原則で対応関係を示す

### まとめ

Natural Mapping は、ユーザーが「考えずに使える」インターフェースを実現するための基本原則です。物理世界での自然な対応関係（例: 上に押せば上に動く）を UI に取り入れることで、学習コストを下げ、エラーを減らすことができます。
