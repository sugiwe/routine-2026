# ユーザーの主導権 (User Control)

- 日付: 2026-01-07
- URL: https://www.sociomedia.co.jp/9058

## メモ（daily より）

システムの都合で強要する操作を減らす。例えば、「これには数分かかることがあります」と表示して何も操作させず待たせる、みたいなのは良くない。

## 深掘り・補足

### コア概念

**ユーザーの主導権**とは、システムがユーザーをコントロールするのではなく、ユーザーがシステムをコントロールできるようにする、というUI設計の基本原則です。

### 3つの基本指針

1. **強制操作の削減**
   - システムの都合による一方的な操作要求を最小化する
   - ユーザーの意図しないタイミングでの割り込みを避ける

2. **行動の自由**
   - ユーザーの選択肢や行動を恣意的に制限しない
   - 必要な時に必要な操作ができる状態を維持する

3. **主体性の確保**
   - ユーザーが自らの意思に基づいて判断・実行できる環境を構築
   - システム主導ではなく、ユーザー主導の体験を設計

### 悪い例

- **Windows アップデート中の画面**
  - 「これには数分かかることがあります」と表示して待たせる
  - 進捗状況が不明確
  - ユーザーは操作が制限され、何もできない
  - いつ終わるかわからない不安を強いる

- **モーダルダイアログの乱用**
  - 他の作業を遮断してしまう
  - ダイアログを閉じないと何もできない

- **自動スクロールやページ遷移**
  - ユーザーが読んでいる最中に勝手に移動する
  - システムのペースを押し付ける

### 良い例

- **非同期処理とバックグラウンド実行**
  - アップデートやダウンロードをバックグラウンドで実行
  - ユーザーは他の作業を継続できる
  - 進捗状況を通知で確認できる

- **モードレス設計**
  - ダイアログではなくインラインで情報を表示
  - 他の部分の操作を妨げない

- **キャンセル可能な操作**
  - 長時間かかる処理でもキャンセルボタンを提供
  - ユーザーが中断したい時に中断できる

- **遅延可能な操作**
  - 「今すぐアップデート」「後で通知」など選択肢を提供
  - ユーザーのペースで実行できる

### 実践的なポイント

1. **UIロック回避**
   - 処理中でも他の部分は操作可能にする
   - プログレスインジケーターで状態を可視化

2. **ユーザーのペース尊重**
   - システムが押し付けるタイミングではなく、ユーザーの速度で進行
   - 自動再生や自動遷移は避けるか、オプションにする

3. **ホットスポット拡張**
   - クリック可能な領域を適切に設計
   - 小さすぎるボタンはユーザーの操作を妨げる

4. **元に戻せる設計**
   - Undo/Redo 機能の提供
   - 安心して操作を試せる環境

### まとめ

ユーザーの主導権を確保することは、使いやすいUIの基本です。システムの都合を優先するのではなく、常に「ユーザーが主役」という視点で設計することが重要です。
