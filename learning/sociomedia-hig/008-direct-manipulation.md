# 直接操作（Direct Manipulation）

- 日付: 2026-01-08
- URL: https://www.sociomedia.co.jp/9075

## メモ（daily より）

画面上のオブジェクトを直接操作しているように感じさせる。大抵 JS が必須になるやつ

## 深掘り・補足

### 概念

「直接操作」とは、ユーザーが画面上のオブジェクトに直接触れて操作しているという感覚を提供する UI デザイン原則です。物理的なオブジェクトを扱うような自然な操作感を実現します。

### 3 つの重要な要素

1. **リアルタイムフィードバック**
   - 身体的な動作に追従してリアルタイムに表示を更新
   - ユーザーの行動に即座に反応することで、直接操作している感覚を強化

2. **可逆性**
   - 逆の操作をすることで前の状態に戻せるようにする
   - ユーザーが安心して試行錯誤できる環境を実現
   - 「元に戻す」機能の重要性

3. **直感性**
   - 物理的なオブジェクト操作と同様の感覚
   - 習得障壁が低い
   - メタファーとの一貫性

### 良い例

- **ドラッグ＆ドロップ**
  - ファイルをフォルダにドラッグして移動
  - カードを並び替える UI
  - リアルタイムに視覚的フィードバックを提供

- **スライダー**
  - 音量調整、明度調整
  - つまみを動かすと即座に結果が反映される

- **タッチジェスチャー**
  - ピンチイン・ピンチアウトで拡大縮小
  - スワイプでページめくり
  - 物理的な動作に近い操作感

- **インライン編集**
  - テキストをクリックしてその場で編集
  - 別画面に遷移せず、オブジェクトに直接働きかける

### 悪い例

- **間接的な操作**
  - オブジェクトを選択 → メニューから操作を選ぶ → ダイアログで設定
  - 複数のステップを経る操作は「直接」感が失われる

- **フィードバックの遅延**
  - ドラッグしても表示が追従しない
  - 操作結果が表示されるまで時間がかかる

- **不可逆な操作**
  - 元に戻せない操作を確認なしで実行
  - ユーザーが試すことをためらう UI

### 実装上の注意点

- **パフォーマンス**: リアルタイムフィードバックには高いパフォーマンスが必要（大抵 JS が必須）
- **アクセシビリティ**: マウス・タッチだけでなく、キーボード操作にも対応
- **モバイル対応**: タッチデバイスでの操作感を最適化
- **視覚的手がかり**: オブジェクトが操作可能であることを明示（カーソル変化、ホバー効果など）

### 関連する原則

- シグニファイア（操作可能性の手がかり）
- フィードバック（操作結果の明示）
- ユーザーコントロール（ユーザーが主導権を持つ）
- 一貫性（物理的メタファーとの整合性）
