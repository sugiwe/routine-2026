# フィッツの法則（Fitts' Law）

- 日付: 2026-01-16
- URL: https://www.sociomedia.co.jp/9146

## メモ（daily より）

パソコンのスクリーンなどで、どの位置が指し示しやすいかの順番を決める法則。「距離」と「アクセスしやすさ」の両方が関わってくる。例えば四隅は「そこまで行けばマウスが止まる」ので、マウス操作においてはかなりアクセスしやすいポイント。タッチデバイスの場合は四隅のアクセスしやすい性質が無くなるので、順位が変わることになる。

感想: 初めて聞く名前の法則だけど、面白い

## 深掘り・補足

### 基本原則

フィッツの法則は、**ターゲットまでの距離と、ターゲットのサイズが、操作時間に影響を与える**という人間工学的原則です。

- **近くて大きい要素** → 素早く正確に選択できる
- **遠くて小さい要素** → 時間がかかり、誤操作も起きやすい

### アクセスしやすさの優先順位

1. **現在のポインター位置**（最も近い）
2. **スクリーンの四隅**（マウスが物理的に止まる）
3. **スクリーンの端**（上下左右）
4. **大きくて近い要素**
5. **小さくて遠い要素**（最も難しい）

### デバイスによる違い

#### マウス操作
- **四隅が特別に有利**: 画面の端まで行けばマウスが物理的に止まるため、正確に狙いやすい
- macOS のメニューバーが画面上端にあるのは、この法則を活用した設計

#### タッチデバイス
- **四隅の優位性が消える**: 画面端で指が止まる効果がない
- 現在の指の位置から近くて大きい要素が最もアクセスしやすい
- 親指でタップしやすい位置（画面下部）が重要になる

### UI 設計への応用

#### 良い例
- **重要なボタンは大きく配置**: 「保存」「送信」などのプライマリアクションは目立つサイズに
- **頻繁に使う機能は近くに**: ユーザーが今いる場所の近くに配置
- **画面端を活用**: macOS のメニューバー、Windows のタスクバーなど

#### 悪い例
- **小さすぎるボタン**: モバイルで 44px 未満のタップターゲット
- **重要な機能が遠い**: ユーザーの作業位置から離れた場所に配置
- **画面端の無駄**: 端の有利性を活かさない配置

### 実践的なポイント

1. **プライマリアクションは大きく**: 最も使われるボタンは視覚的にもサイズ的にも優遇する
2. **関連する機能は近くに配置**: ワークフローに沿った配置で移動距離を短縮
3. **タッチターゲットは最低 44×44px**: iOS のヒューマンインターフェースガイドラインでも推奨
4. **デバイスに応じた配置**: マウス操作とタッチ操作で最適な配置が異なることを理解する

### 関連する原則

- **直接操作（Direct Manipulation）**: 対象に直接触れる感覚でアクセスしやすさが向上
- **視覚的ゲシュタルト（Visual Gestalt）**: 関連する要素を近くにまとめることで移動距離を短縮
