# タスクコヒーレンス (Task Coherence)

- 日付: 2026-01-19
- URL: https://www.sociomedia.co.jp/9194

## メモ（daily より）

ユーザーが、昨日行ったことを今日も行う可能性が高いことを、タスクコヒーレンスという。アプリケーション再起動時に、前回終了したタスクを引き継いで再表示しておくことでユーザーにすぐ続きを始められるようにできる。また、Finder の「最近使ったファイル」もタスクコヒーレンス的に有効な機能。

**感想**: タスクコヒーレンスは初めて聞く用語だったけど納得感高い話だった。

## 深掘り・補足

### タスクコヒーレンスとは

タスクコヒーレンスは、「ユーザーが昨日行ったことを今日も行う可能性は高い」という原則に基づく UI デザインの考え方です。ユーザーの過去の行動パターンを記憶し、それを活用することで、将来の行動を効果的に予測し、スムーズな作業継続を支援します。

### 実装方法

タスクコヒーレンスを実現する主な方法は以下の 3 つです：

#### 1. 前回状態の復元

アプリケーション再起動時に、前回終了した時点のタスクや画面状態を引き継ぎます。

**具体例**：
- テキストエディタが前回開いていたファイルとカーソル位置を復元
- ブラウザが前回のタブとスクロール位置を復元
- 音楽プレイヤーが前回の再生位置から再開

#### 2. 状態保存

画面の状態やユーザー設定を記録し、次回起動時に再現することで継続性を確保します。

**具体例**：
- ウィンドウのサイズと位置
- 表示モード（リスト表示／グリッド表示など）
- フィルタやソート設定
- 入力フォームの途中データ

#### 3. 最近使用項目の表示

「最近開いた項目」メニューなどを提示し、ユーザーがすぐに作業を再開できるようにします。

**具体例**：
- Finder の「最近使ったファイル」
- アプリケーションの「最近開いたファイル」メニュー
- Web サービスの「閲覧履歴」「購入履歴」
- IDE の「最近のプロジェクト」

### 実装効果

タスクコヒーレンスを実装することで、以下の効果が得られます：

- **作業効率の向上**: ユーザーは保存された状態から即座に作業を続けられる
- **認知負荷の軽減**: 「どこまでやったっけ？」と思い出す必要がない
- **利便性の向上**: 毎回同じ設定やファイルを開く手間が省ける
- **離脱率の低下**: 作業再開のハードルが下がり、継続利用が促進される

### 設計時の注意点

#### プライバシーへの配慮

- 共有端末では履歴を残さないオプションを提供
- センシティブな情報（パスワード入力画面など）は復元しない
- ユーザーが履歴をクリアできる機能を提供

#### パフォーマンスとのバランス

- 状態保存のタイミングを適切に設定（過度な保存は動作を重くする）
- 復元するデータ量を適切に制限（すべてを復元すると起動が遅くなる）

#### ユーザーの意図を尊重

- 新規タスクを開始したい場合もあるため、「新規で開始」オプションも提供
- 予期しない状態復元でユーザーを混乱させないよう、復元内容を明示

### 関連する原則

- **一貫性 (Consistency)**: 状態保存・復元の挙動が予測可能であること
- **ユーザーコントロール (User Control)**: 状態復元の有無をユーザーが選択できること
- **エラー回避 (Avoid Errors)**: 誤って閉じてしまった場合でも作業を復元できること

### まとめ

タスクコヒーレンスは、ユーザーの行動パターンを活用してスムーズな作業継続を支援する重要な原則です。適切に実装することで、ユーザーエクスペリエンスを大きく向上させることができます。ただし、プライバシーやパフォーマンスへの配慮も忘れずに実装することが重要です。
