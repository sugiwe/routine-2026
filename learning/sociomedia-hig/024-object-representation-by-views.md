# ビューはオブジェクトを表象する (Object Representation by Views)

- 日付: 2026-01-24
- URL: https://www.sociomedia.co.jp/9256

## メモ（daily より）

インターフェースは、オブジェクト（ユーザーの関心の対象）を表象するビューの集まりによって構成される。またひとつのオブジェクトは複数のビューによって異なる表現を持つことができる。オブジェクトは、一覧、詳細、アイコンなど複数のビュー（ひとまとまりの表示要素）として画面上に提示され、それらの組み合わせでユーザーが知覚するシステムの全体像が作られる。

## 深掘り・補足

### 基本原則

インターフェース設計において、**オブジェクト（ユーザーの関心の対象）をビューで表象する**という考え方は、UI/UX 設計の根幹となる重要な概念です。

> インターフェースは、オブジェクトを表象するビューの集まりによって構成される

### 重要なポイント

#### 1. 同一オブジェクトの複数ビュー

ひとつのオブジェクトは、利用シーンや目的に応じて異なる表現形式で提示できます：

- **一覧ビュー**: 複数のオブジェクトを俯瞰的に把握
- **詳細ビュー**: ひとつのオブジェクトの詳細情報を表示
- **アイコン**: コンパクトな視覚的表現
- **サムネイル**: 画像やプレビューで内容を視覚的に把握
- **カード形式**: 概要情報と操作を含む中間的な表現

#### 2. オブジェクト中心のインターフェース設計

ユーザーが関心を持つ「オブジェクト」を中心に UI を構成することで：

- ユーザーの認知負荷を軽減
- 直感的な操作を実現
- システム全体の一貫性を保つ

#### 3. ビューの組み合わせによる全体像

各ビューは独立した要素ではなく、それらの組み合わせによってユーザーが知覚するシステムの全体像が作られます。

### 実践例

#### ファイル管理システム

同じ「ファイル」というオブジェクトを：

- **リストビュー**: ファイル名、サイズ、更新日時を一覧表示
- **アイコンビュー**: ファイルタイプを視覚的に表現
- **プレビュービュー**: ファイルの内容を直接確認
- **詳細ビュー**: メタデータや権限などの詳細情報

#### メールアプリケーション

「メール」というオブジェクトを：

- **受信トレイ（一覧）**: 差出人、件名、日時
- **メール本文（詳細）**: 完全な内容と添付ファイル
- **スレッドビュー**: 会話の流れとして表現
- **通知バッジ**: 未読数をコンパクトに表示

#### EC サイトの商品

「商品」というオブジェクトを：

- **検索結果（一覧）**: サムネイル、価格、評価
- **商品詳細ページ**: 写真、説明、レビュー、在庫状況
- **カート内表示**: 商品名、数量、小計
- **注文履歴**: 購入日、配送状況

### 設計時の考慮点

1. **ビュー間の一貫性**: 同一オブジェクトであることが認識できるよう、一貫した表現を維持
2. **文脈に応じた最適化**: 各ビューは利用場面に最適化された情報を提供
3. **スムーズな遷移**: ビュー間の移動が自然で予測可能であること
4. **情報の段階的開示**: 概要から詳細へ、段階的に情報を提示

### 関連する原則

- **オブジェクトベースにする**: UI をオブジェクト中心に構成
- **名詞→動詞の操作順序**: オブジェクトを選択してから操作を指定
- **一貫性の原則**: 同一オブジェクトは一貫した表現を保つ

### まとめ

ビューはオブジェクトを表象するという原則は、ユーザーが何に関心を持ち、どのように情報を知覚するかに焦点を当てた設計思想です。同一のオブジェクトを状況に応じて最適な形で提示することで、直感的で使いやすいインターフェースを実現できます。
