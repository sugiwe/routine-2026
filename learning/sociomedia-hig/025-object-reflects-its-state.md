# オブジェクトは自身の状態を体現する (Object Reflects Its State)

- 日付: 2026-01-25
- URL: https://www.sociomedia.co.jp/9267

## メモ（daily より）

画面上のオブジェクトは、自身の状態を視覚的かつリアルタイムに体現し続ける必要がある

## 深掘り・補足

### 原則の意味

アイコン、選択可能項目、各種コントロール、ペインやウィンドウなど、画面上のすべてのオブジェクトは、**自身の状態を視覚的かつリアルタイムに体現し続けなければならない**という UI デザインの基本原則です。

### なぜ重要か

ユーザーはインタラクティブな操作を通じてオブジェクトの状態を変化させながら、作業を目標に近づけていきます。オブジェクトが自身の状態を明確に示すことで：

- **即座のフィードバック**：ユーザーは操作の結果をすぐに理解できる
- **現在地の把握**：システムの現在状態を視覚的に確認できる
- **直感的な操作**：状態が見えることで次の行動を判断しやすい
- **エラーの防止**：意図しない状態変化に気づきやすい

### 具体例

ファイルアイコンの状態表現：

1. **通常状態**：標準の表示
2. **選択時**：ハイライト表示で選択されていることを示す
3. **ドラッグ中**：半透明や移動中の視覚効果
4. **ロック状態**：鍵アイコンなどで編集不可を示す
5. **不可視ファイル**：薄い表示や特殊なマークで隠しファイルを示す
6. **ダウンロード進行中**：プログレスバーやパーセント表示
7. **プレビュー表示時**：拡大表示や枠線で注目状態を示す

### 実装のポイント

#### 良い例

- ボタンが押せない状態の時はグレーアウト（disabled 状態）
- チェックボックスのオン/オフが視覚的に明確
- ドラッグ中のアイテムが半透明になり、ドロップ先がハイライトされる
- ローディング中はスピナーやプログレスバーを表示
- 保存済み/未保存の状態をアイコンやテキストで明示

#### 悪い例

- ボタンが押せない状態でも見た目が変わらない
- 選択されているかどうかが分かりにくい
- 処理中なのか完了したのか判断できない
- エラー状態でも通常時と同じ表示

### 関連する原則

- **視認性（Visibility）**：重要な情報は目に見える形で提供する
- **フィードバック**：ユーザーの操作に対して即座に反応を返す
- **システムステータスの可視化**：システムの現在状態を常に明示する
