# グラフィックのトーン＆マナーを揃える (Tone and Manner)

- 日付: 2026-01-27
- URL: https://www.sociomedia.co.jp/9282

## メモ（daily より）

画面全体・アプリ全体で見た目の雰囲気を揃える

## 深掘り・補足

### トーン＆マナーとは

グラフィック要素全体の視覚的な統一性を指す概念です。具体的には、以下の要素の度合いを揃えることを意味します：

- **色彩関連**：色調、彩度、明度
- **装飾効果**：グラデーションのつけ方、影のつけ方、ボーダーのつけ方
- **形状**：角の取り方（角丸 vs 直角）
- **表現スタイル**：塗りと線のあり方、線の太さ
- **表現方法**：カラー vs シルエット、具象的 vs 抽象的

### なぜ重要なのか

トーン＆マナーを揃えることは、**UI デザインにおける一貫性の原則**を視覚的に実現する手段です。統一された視覚表現により：

1. **認知負荷の軽減**：ユーザーが画面を理解しやすくなる
2. **プロフェッショナルな印象**：洗練された統一感のある UI に見える
3. **ブランドアイデンティティの強化**：アプリ全体で一貫した印象を与える
4. **操作の予測可能性**：視覚的パターンが揃うことで、次の操作を予測しやすくなる

### 実践的なガイドライン

#### 良い例

- すべてのボタンが同じ角丸の半径を持つ
- アイコンのスタイルが統一されている（すべてフラット、またはすべて立体的）
- 影の効果が同じパラメータ（距離、ぼかし、不透明度）で適用されている
- カラーパレットが統一されており、同じ彩度・明度のレベルで使用されている

#### 悪い例

- あるボタンは角丸で、別のボタンは直角
- 一部のアイコンは具象的でリアル、他は抽象的でシンプル
- ある要素には濃い影、別の要素には薄い影またはグラデーション
- 色のトーンがバラバラ（一部は鮮やか、一部はくすんでいる）

### デザインスペクトラムの選択

トーン＆マナーを決める際、以下のようなスペクトラムから一貫した位置を選ぶ必要があります：

- **フラット ←→ 立体的**
- **抽象的 ←→ 具象的**
- **シンプル ←→ 装飾的**
- **モノクロ/シルエット ←→ カラフル**
- **太い線 ←→ 細い線**

重要なのは、スペクトラムのどこを選ぶかではなく、**選んだ位置を全体で統一すること**です。

### 関連する原則

- **一貫性（Consistency）**：トーン＆マナーは一貫性を実現する具体的手段
- **色やフォントを使いすぎない**：統一されたパレットの範囲内で作業する
- **整然とレイアウトする**：視覚的統一性はレイアウトの整然さにも寄与する

### 実装のヒント

1. **デザインシステムの構築**：トーン＆マナーのルールをドキュメント化し、チーム全体で共有
2. **コンポーネントライブラリの活用**：統一されたスタイルのコンポーネントを再利用
3. **定期的なデザインレビュー**：新しい要素が既存のトーン＆マナーから逸脱していないか確認
4. **スタイルガイドの作成**：具体的な数値（角丸の半径、影のパラメータなど）を明記
