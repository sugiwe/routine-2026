# ユーザーが入力したものはユーザーのもの (User Owns Inputs)

- 日付: 2026-02-07
- URL: https://www.sociomedia.co.jp/9398
- daily 記録: [daily/2026-02-07.md](../../daily/2026-02-07.md)

## ユーザーの第一印象（daily より）

ユーザーが入力した値や設定した内容は基本的にすべて保存されるべき。ユーザーが追加した項目はユーザーが削除できなければいけない。ユーザーが入力した内容はユーザーが変更できなければいけない。

## 深掘り・補足

### 基本原則

「ユーザーが入力したものはユーザーのもの」という原則は、ユーザーデータの所有権と管理に関する重要な設計指針です。この原則には 3 つの主要な要件があります：

1. **保存義務**: ユーザーが入力・設定した全データは保存される必要がある
2. **削除権**: ユーザーが追加した項目は、ユーザー自身が削除できなければならない
3. **編集権**: ユーザーが入力した内容は、ユーザーが変更・修正できなければならない

### なぜこの原則が重要なのか

- **ユーザーの主体性**: ユーザーが自分のデータに対する完全な支配権を持つことで、システムへの信頼が生まれる
- **データの透明性**: 何が保存され、どう管理されるかが明確になる
- **エラーの修正**: ユーザーが間違って入力した内容を自分で修正できる

### 良い例

- **Amazon のアカウント設定**: ユーザーが名前や住所などの個人情報を自由に変更できる
- **メモアプリ**: ユーザーが作成したメモを自由に編集・削除できる
- **SNS の投稿**: 自分が投稿した内容を後から編集・削除できる

### 悪い例

- ユーザーが入力したデータが勝手に削除される（セッション切れなど）
- ユーザーが追加した項目を削除する手段がない
- 管理者しかデータを変更できず、ユーザー自身は変更できない
- 入力内容が保存されず、画面遷移で消えてしまう

### 実装上の注意点

- **自動保存**: フォーム入力中のデータを自動的に一時保存する
- **明確な削除UI**: ユーザーが簡単に削除操作を見つけられるようにする
- **編集の容易さ**: 編集モードへの切り替えがスムーズであること
- **確認メッセージ**: 削除などの重要な操作には確認ダイアログを表示する（ただし過剰にならないように）

### 関連する原則

- **Forgiveness（寛容性）**: ユーザーのミスを許容し、元に戻せるようにする
- **Transparency（透明性）**: システムの状態や動作をユーザーに明確に伝える
- **User Control（ユーザーコントロール）**: ユーザーが主導権を持ち、システムを操作できる
