# 整合性を損なうような操作をユーザーに求めない (Don't Let Inconsistent Input)

- 日付: 2026-02-08
- URL: https://www.sociomedia.co.jp/9408
- daily 記録: [daily/2026-02-08.md](../../daily/2026-02-08.md)

## ユーザーの第一印象（daily より）

情報の整合性が損なわれる可能性がある操作をユーザーに求めない。ブレーキランプを手動でオンにさせてはいけない。例えば、生年月日と年齢を両方入力させるようなことはしない。

### 感想

出てくる例（生年月日と年齢をどちらも記入させる）は、紙の書類では結構普通に発生していていつも不便だなと思っていた。デジタルなら仕組みで解決できるので素晴らしいなと思った。

## 深掘り・補足

### この原則の核心

**情報の整合性を維持する責任は、システム側が負うべき**という考え方です。ユーザーに矛盾した情報や冗長な情報を入力させることは、以下の問題を引き起こします：

1. **データの矛盾リスク**：生年月日と年齢が一致しない、など
2. **ユーザーの負担増加**：同じ情報を複数の形式で入力する手間
3. **エラーの温床**：手動入力による計算ミスや誤記
4. **信頼性の低下**：「このシステムは賢くない」という印象

### 良い例と悪い例

#### 悪い例

```
生年月日: [____年__月__日]
年齢: [__歳]  ← 両方入力させるのは NG
```

```
開始時刻: [10:00]
終了時刻: [12:00]
所要時間: [__時間__分]  ← 計算可能なのに入力させるのは NG
```

#### 良い例

```
生年月日: [____年__月__日]
（年齢は自動計算されて表示のみ）
```

```
開始時刻: [10:00]
終了時刻: [12:00]
所要時間: 2時間 0分  ← 自動計算して表示
```

### ブレーキランプのメタファー

ブレーキランプは、ドライバーがブレーキペダルを踏むと**自動的に点灯**します。もし手動でブレーキランプを操作しなければならないとしたら：

- ブレーキを踏みながらランプのスイッチも押す必要がある
- 操作を忘れたり、タイミングがずれたりする
- 後続車に正しい情報が伝わらない危険性

これと同じように、**システムが自動的に導き出せる情報をユーザーに手動入力させるべきではない**のです。

### 実践的なアドバイス

1. **導出可能な情報は自動計算する**
   - 生年月日 → 年齢
   - 開始・終了時刻 → 所要時間
   - 単価 × 数量 → 小計

2. **選択の矛盾を防ぐ**
   - 「配送先住所」と「請求先住所」が同じ場合は「同じ住所を使用」チェックボックスを提供
   - 重複入力を避ける

3. **入力の整合性をシステムで保証する**
   - 日付の範囲チェック（開始日 ≦ 終了日）
   - 論理的な矛盾をリアルタイムで検証

4. **デフォルト値で賢くサポート**
   - よく使われる値を自動的に設定
   - 前回の入力内容を記憶

### デジタルならではの利点

紙の書類では、情報の整合性を保つために同じ情報を複数記入させることがありました（役所の書類など）。しかし、デジタルでは：

- **一度入力すれば、システムが他の形式に自動変換できる**
- **リアルタイムで矛盾をチェックできる**
- **計算や導出を瞬時に行える**

この利点を最大限に活かすことが、優れたデジタル体験につながります。

### 関連する原則

- **Automate The Single Option**（選択肢が 1 つしかないなら自動化する）
- **Precomputation**（事前計算）
- **Avoid Errors**（エラーを避ける）
- **Data Binding**（データをバインドする）

### まとめ

ユーザーに整合性の維持を任せるのではなく、**システムが責任を持って整合性を保証する**ことが重要です。これにより、ユーザーの負担が減り、データの品質が向上し、全体的なユーザー体験が改善されます。
