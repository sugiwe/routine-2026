# 40. 入力フォームにはストーリー性を持たせる (Give Form A Story)

- 日付: 2026-02-09
- URL: https://www.sociomedia.co.jp/9421
- daily 記録: [daily/2026-02-09.md](../../daily/2026-02-09.md)

## ユーザーの第一印象（daily より）

入力項目はユーザーにとって意味のあるグループにまとめる。項目の並び順は、ユーザーにとって身近なことや単純なものを先にし、複雑なものを後にする。

## 深掘り・補足

### 中核となる考え方

フォーム設計において「入力項目はユーザーにとって意味のあるグループにまとめる」ことが重要です。単なるランダムな配置ではなく、ユーザーの思考フローに沿った構成を心がけましょう。

### 項目の並び順の原則

「ユーザーにとって身近なことや単純なものを先にし、複雑なものを後にする」という流れを作ることで、回答のしやすさが大幅に向上します。これにより、ユーザーは段階的にフォームに集中でき、途中で離脱するリスクが減少します。

### 良い例と悪い例

**良い例（ストーリー性あり）：**
```
名前 → 自宅電話 → 自宅住所 → 会社名 → 会社電話 → 会社住所
```
- 個人情報から始まり、次に会社情報へ進む自然な流れ
- 関連する情報がグループ化されている
- ユーザーの記憶の引き出し方に沿っている

**悪い例（ストーリー性なし）：**
```
自宅住所 → 会社住所 → 自宅電話 → 会社電話 → 名前 → 会社名
```
- 住所と電話番号がバラバラに配置されている
- 自宅情報と会社情報が混在している
- ユーザーが情報を思い出す際に混乱を招く

### 実践的なポイント

1. **個人情報は段階的に進める**
   - 基本情報（名前、メールアドレス）から始める
   - 徐々に詳細な情報（住所、電話番号）へ進む
   - 最後に機密性の高い情報（クレジットカード情報など）を配置

2. **関連情報をグループ化する**
   - 自宅情報：住所、電話番号、郵便番号
   - 会社情報：会社名、部署、役職、会社住所
   - 支払い情報：クレジットカード番号、有効期限、セキュリティコード

3. **ユーザーの自然な思考順序を追う**
   - 頭の中で情報を引き出す順序を考慮
   - 一般的な情報から特定の情報へ
   - 覚えやすい情報から覚えにくい情報へ

4. **心理的な負荷を段階的に増加させる設計**
   - 最初は簡単な質問で心理的ハードルを下げる
   - 徐々に詳細な質問へ移行
   - ユーザーの投資意識を高めることで完了率を向上

### 関連する原則

- **認知負荷の軽減**: 情報をグループ化することで、ユーザーの認知負荷を減らす
- **進捗の可視化**: ストーリー性のある構成により、ユーザーはフォームの進捗を理解しやすくなる
- **コンテキストの維持**: 関連する情報をまとめることで、ユーザーは現在のコンテキストを維持しやすくなる
