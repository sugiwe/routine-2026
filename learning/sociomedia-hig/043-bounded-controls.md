# 43. 制限コントロールを活用する (Bounded Controls)

- 日付: 2026-02-12
- URL: https://www.sociomedia.co.jp/9446
- daily 記録: [daily/2026-02-12.md](../../daily/2026-02-12.md)

## ユーザーの第一印象（daily より）

既定項目の選択、数値入力、日付入力など、入力可能な内容に制限がある場合は、ラジオボタン、ドロップダウンメニュー、ステッパー、スライダー、ピッカー、などの制限コントロールを用いて有効な値のみが入力可能となるようにする。

## 深掘り・補足

### コアコンセプト

制限コントロール（Bounded Controls）は、**ユーザーに自由なテキスト入力を許可するのではなく、あらかじめ用意された有効な選択肢から選ばせる**設計パターンです。これにより、入力エラーを防止し、データの整合性を保ちます。

### なぜ重要か

1. **入力エラーの防止**
   - 不正な値の入力を物理的に不可能にする
   - バリデーションエラーによるユーザーのフラストレーションを減らす

2. **認知負荷の軽減**
   - ユーザーは「何を入力すべきか」を考える必要がない
   - 選択肢を見ることで、入力可能な範囲を直感的に理解できる

3. **データ品質の向上**
   - システムが期待する形式で必ずデータが入力される
   - 後処理やクレンジングの手間が削減される

### 具体的なコントロールと使い分け

| コントロール | 適した用途 | 選択肢数の目安 |
|------------|----------|--------------|
| **ラジオボタン** | 少数の選択肢から1つを選ぶ | 2〜5個 |
| **ドロップダウンメニュー** | 多数の選択肢から1つを選ぶ | 6個以上 |
| **ステッパー** | 連続した数値の増減 | 小範囲の数値 |
| **スライダー** | 範囲内での数値選択 | 厳密さを要求しない数値 |
| **ピッカー** | 日付・時刻・リストから選択 | 日付、時刻、構造化データ |

### 良い例と悪い例

#### ❌ 悪い例：自由入力

```
生年月日を入力してください: [__________]
→ ユーザーは「2024/1/5」「2024-01-05」「令和6年1月5日」など様々な形式で入力
→ バリデーションエラー、形式統一の手間
```

#### ✅ 良い例：制限コントロール

```
生年月日を選択してください:
[年 ▼] [月 ▼] [日 ▼]
→ 有効な日付のみ選択可能
→ システムが期待する形式で確実に入力される
```

#### ❌ 悪い例：不適切なコントロール選択

```
年齢: [___] ← 自由入力（「25歳」「二十五」など様々な入力）
```

#### ✅ 良い例：適切なコントロール

```
年齢: [18 ▼] （ドロップダウンで18〜100）
または
年齢: [25] [- +] （ステッパーで増減）
```

### 関連する設計原則

- **よいデフォルト値の提供**（Good Defaults）
  - 制限コントロールと組み合わせることで、さらにユーザー負荷を軽減

- **値入力ではなく結果選択**（Select Results, Not Values）
  - ユーザーが求める「結果」に基づいた選択肢を提供

- **ユーザーに厳密さを求めない**（Don't Require Precision）
  - スライダーなど、厳密な値が不要な場合は柔軟な入力方法を

### 注意点

1. **選択肢が多すぎる場合**
   - ドロップダウンが100項目を超える場合は検索機能付きのセレクトボックスを検討
   - オートコンプリート付きテキスト入力も選択肢の一つ

2. **柔軟性とのバランス**
   - すべてを制限すると、イレギュラーな入力が必要な場合に対応できない
   - 「その他」オプション + 自由入力欄などの逃げ道を用意することも重要

3. **アクセシビリティ**
   - スクリーンリーダーでも操作可能なコントロールを選択
   - キーボードだけでも操作できるように設計

### まとめ

制限コントロールは、**ユーザーエクスペリエンスとデータ品質の両立**を実現する強力な設計手法です。ただし、適切なコントロールの選択と、必要に応じた柔軟性の確保が成功の鍵となります。
