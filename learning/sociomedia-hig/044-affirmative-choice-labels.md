# 選択肢の文言は肯定文にする

- 日付: 2026-02-13
- URL: https://www.sociomedia.co.jp/9456
- daily 記録: [daily/2026-02-13.md](../../daily/2026-02-13.md)

## 深掘り・補足

### なぜ肯定文にするべきか

否定形のラベルは、ユーザーに二重の認知負荷をかけます：

1. **否定文の理解**: まず否定文そのものを理解する必要がある
2. **選択との組み合わせ**: チェックを入れることで「否定を肯定する」という矛盾した意味になる

例えば、「新幹線以外の有料特急を利用しない」というチェックボックスがあった場合：

- チェックを入れる = 利用しないことを選択する = 結局利用するの？しないの？

### 具体的な改善例

**悪い例：**

```
□ 新幹線以外の有料特急を利用しない
□ メールマガジンを受信しない
□ 通知を表示しない
```

**良い例：**

```
□ 新幹線のみ利用
□ 有料特急を利用
□ 飛行機を利用

または

☑ メールマガジンを受信する
☑ 通知を表示する
```

### 設計のポイント

1. **選択肢は何を有効にするかで表現する**
   - 「〜しない」ではなく「〜する」
   - 「〜を除外」ではなく「〜を含める」

2. **デフォルト状態を明確にする**
   - 肯定文にすることで、チェックが入っている=その機能が有効、という直感的な理解が可能

3. **例外的に否定文を使う場合**
   - 確認ダイアログなど、明示的に「実行しない」という選択が重要な場面
   - ただしこの場合もボタンラベルは「キャンセル」「いいえ」など単純な否定語を使う

### UIパターンごとの考慮事項

**チェックボックス:**

- チェックを入れる = その項目を有効にする
- 肯定文が最も重要

**ラジオボタン:**

- 複数の肯定的な選択肢から1つを選ぶ形式が理想
- 「〜しない」という選択肢が必要な場合は「利用しない」「設定しない」など単純な表現に

**トグルスイッチ:**

- オン = 機能が有効、オフ = 機能が無効
- ラベルは必ず機能の名前（肯定形）で表現

### ユーザーの混乱を防ぐ

否定文を使うと、特に急いでいるときや注意力が散漫なときに、ユーザーは逆の意味に解釈してしまう可能性があります。肯定文を使うことで、このようなエラーを防ぎ、よりスムーズなユーザー体験を提供できます。
