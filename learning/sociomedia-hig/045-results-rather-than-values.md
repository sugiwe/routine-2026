# 45. 値を入力させるのではなく結果を選ばせる (Results Rather Than Values)

- 日付: 2026-02-14
- URL: https://www.sociomedia.co.jp/9464
- daily 記録: [daily/2026-02-14.md](../../daily/2026-02-14.md)

## ユーザーの第一印象（daily より）

自分で細かいパラメーターを設定していくよりも、選択肢から選ぶことを好むユーザーが多い。その通りな気もするし、最終的に細かい調整をしたくなる気もする。

## 深掘り・補足

### 核となる原則

「多くの場合ユーザーは自分で入力するよりも選択肢から選ぶことを好む」というのがこの HIG の中心的な考え方です。パラメータの数値を手入力させるのではなく、その結果を視覚的に提示して選ばせる方が、ユーザー体験として優れています。

### 具体例: Adobe Photoshop

- **色調バリエーション機能**: 結果のサムネールを選択する方式 → ユーザーフレンドリー ✅
- **カラーバランス機能**: 数値入力を要求する方式 → より難易度が高い

この違いは、「結果を見て選ぶ」ことの方が「数値から想像する」ことよりも直感的であることを示しています。

### 設計上の推奨事項

#### 良い例

- 複数の処理結果を並べて表示し、ユーザーが視覚的に比較できる
- サムネイルやプレビューで「どうなるか」を先に見せる
- スライダーと連動したリアルタイムプレビュー

#### 避けるべき例

- 抽象的なパラメータ（彩度、明度など）を数値で入力させる
- 結果を確認する前に複雑な設定を要求する

### ユーザーの視点から

ユーザーの第一印象にあった「最終的に細かい調整をしたくなる気もする」という点は非常に重要な洞察です。理想的な UI は:

1. **初期段階**: 結果を並べて選ばせる（素早く大枠を決定）
2. **詳細調整**: 必要に応じて数値での微調整も可能にする

この2段階アプローチにより、初心者は簡単に使え、上級者は細かくコントロールできるという両立が可能になります。

### 実装のヒント

- **プリセット + カスタマイズ**: まずプリセットから選び、後から細かく調整できる設計
- **ライブプレビュー**: パラメータを変更すると即座に結果が反映される
- **比較表示**: Before/After や複数候補の並列表示

この原則は、認知的負荷を減らし、ユーザーが「試行錯誤」ではなく「選択」によって目的を達成できるようにすることを目指しています。
