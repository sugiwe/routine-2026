# 47. デフォルトボタンには具体的な動詞を用いる (Specific Verb For Default Button Title)

- 日付: 2026-02-16
- URL: https://www.sociomedia.co.jp/9486
- daily 記録: [daily/2026-02-16.md](../../daily/2026-02-16.md)

## ユーザーの第一印象（daily より）

「キャンセルのキャンセル」の選択がどっちかわからないというのはたまに見かけるやつ

## 深掘り・補足

### 原則の核心

モーダルダイアログやフォーム画面のデフォルトボタン（主要なアクションボタン）には、「OK」や「はい」といった汎用的な言葉ではなく、**実際に実行されるアクションを表す具体的な動詞**を使用すべきです。

### なぜ重要か

ユーザーは確認ダイアログを読む際、特に急いでいる時や慣れた操作の場合、メッセージ本文を十分に読まずにボタンだけを見て判断することがあります。この時、ボタンラベルが曖昧だと：

- **誤操作のリスク**: 意図しない操作を実行してしまう
- **認知的負荷の増加**: メッセージとボタンの関連性を頭の中で変換する必要がある
- **ユーザーの不安**: 「このボタンを押して本当に大丈夫か？」という迷いが生じる

### 良い例と悪い例

#### 悪い例
```
ファイルを削除しますか？
[OK] [キャンセル]
```
問題点：
- 「OK」が何を意味するのか、メッセージを読み返さないと分からない
- 「OK」は削除の承認なのか、ダイアログを閉じるだけなのか曖昧

#### 良い例
```
ファイルを削除しますか？
[削除] [キャンセル]
```
改善点：
- 「削除」ボタンを押すと削除されることが一目瞭然
- ユーザーは迷わず判断できる

#### 「キャンセルのキャンセル」問題の例

ユーザーが指摘した「キャンセルのキャンセル」は、以下のような状況で発生します：

##### 悪い例
```
キャンセルしますか？
[はい] [いいえ]
```
混乱の原因：
- 「はい」→「キャンセルする」ことへの「はい」なのか？
- 「いいえ」→「キャンセルしない」ことへの「いいえ」なのか？
- 二重否定的な構造で認知的負荷が高い

##### 良い例
```
この操作をキャンセルしますか？
[操作をキャンセル] [作業を続ける]
```
または
```
変更を破棄して戻りますか？
[破棄して戻る] [編集を続ける]
```

### キャンセルボタンの扱い

興味深いことに、**キャンセルボタン自体は「キャンセル」という汎用的な表現で問題ない**とされています。これは：

- 「キャンセル」は「何もせずにダイアログを閉じる」という共通認識がある
- 主要アクションではなく、安全な「戻る」操作を表す
- 一貫性を保つことで学習効果が高まる

ただし、文脈によっては「編集を続ける」「変更しない」など、より具体的な表現の方が親切な場合もあります。

### 実践的なガイドライン

1. **アクションを明確に**: ボタンラベルだけで結果が分かるようにする
2. **動詞を使う**: 「保存」「削除」「送信」「適用」など
3. **二重否定を避ける**: 「〜しないことをキャンセルする」のような構造は避ける
4. **長すぎない**: 具体的であっても、簡潔さは重要
5. **一貫性を保つ**: アプリ全体で似たアクションには同じ言葉を使う

### まとめ

このガイドラインは小さな改善に見えますが、ユーザーの自信と操作スピードに大きく影響します。特に重要な操作（削除、送信、購入など）では、具体的な動詞を使うことでユーザーエラーを防ぎ、より良いUXを提供できます。
