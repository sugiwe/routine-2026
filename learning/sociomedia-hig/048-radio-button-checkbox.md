# 48. ラジオボタンは単数選択、チェックボックスはオンオフ

- 日付: 2026-02-17
- URL: https://www.sociomedia.co.jp/9492
- daily 記録: [daily/2026-02-17.md](../../daily/2026-02-17.md)

## ユーザーの第一印象（daily より）

ラジオボタンはデフォルト必須ってのは知らなかった。

## 深掘り・補足

### ラジオボタンとチェックボックスの使い分け原則

フォーム設計において最も基本的でありながら、しばしば混同されるのがラジオボタンとチェックボックスの使い分けです。この2つのコントロールは見た目が似ていますが、**目的と動作が根本的に異なります**。

### ラジオボタン（Radio Button）

**用途**: 選択肢からの**単数選択（排他的選択）**

**特徴**:
- 「A か B か」という二者択一、または複数の選択肢から1つだけを選ぶ場合に使用
- グループ内で1つだけが選択可能（排他的）
- **必ずデフォルト値を設定する必要がある**
- 一度選択すると、他の選択肢を選ぶことでしか解除できない

**デフォルト値が必須な理由**:
1. ユーザーが「選択しない」という選択肢を持てない設計であるため
2. デフォルトがないと、ユーザーが必ず1つ選ぶ手間が発生し、フォーム入力の負担が増加
3. 「最も一般的な選択肢」をデフォルトにすることで、多くのユーザーの入力手間を削減できる

### チェックボックス（Checkbox）

**用途**: 各項目が独立した**オン・オフ機能**、または**複数選択**

**特徴**:
- 「真か偽か」という状態判定（例: 「利用規約に同意する」）
- 複数の項目を独立して選択・解除可能
- デフォルトはオフ（未選択）でも構わない
- クリックするたびにオン・オフが切り替わる

### 良い例と悪い例

#### ❌ 悪い例

```
性別を選択してください:
☑ 男性
☑ 女性
☑ その他
```

**問題点**: 性別は排他的な選択（1つだけ選ぶべき）なのに、チェックボックスを使っているため、複数選択が可能に見えてしまう。

#### ✅ 良い例

```
性別を選択してください:
◉ 男性
○ 女性
○ その他
```

**理由**: ラジオボタンを使用し、デフォルトで「男性」が選択されている。ユーザーは他の選択肢を選ぶだけで済む。

---

#### ❌ 悪い例

```
通知設定:
◉ メール通知を受け取る
○ メール通知を受け取らない
```

**問題点**: オン・オフの二値選択にラジオボタンを使用している。これはチェックボックスで表現すべき。

#### ✅ 良い例

```
通知設定:
☑ メール通知を受け取る
```

**理由**: 単純なオン・オフ判定なので、チェックボックス1つで十分。ユーザーはチェックを入れるか外すかで直感的に設定できる。

### 実践的なポイント

1. **「AかBか」ならラジオボタン、「真か偽か」ならチェックボックス**
   - 支払い方法: クレジットカード／銀行振込 → ラジオボタン
   - ニュースレター購読の可否 → チェックボックス

2. **ラジオボタンは必ずデフォルト値を設定する**
   - 最も一般的な選択肢、または推奨される選択肢をデフォルトにする
   - 「選択してください」という空の選択肢は避ける

3. **チェックボックスはデフォルトをオフにしても良い**
   - ただし、「よいデフォルト」の原則（HIG #42）に従い、ユーザーにとって最適なデフォルト状態を検討する

4. **二者択一でもチェックボックスが適切な場合がある**
   - 「利用規約に同意する」のような、明示的な意思表示が必要な場合
   - オン・オフの切り替えが直感的に理解できる場合
