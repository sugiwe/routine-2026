# 52. スクロール画面では続きがありそうに見せる (Avoid The False Bottom)

- 日付: 2026-02-21
- URL: https://www.sociomedia.co.jp/9519
- daily 記録: [daily/2026-02-21.md](../../daily/2026-02-21.md)

## ユーザーの第一印象（daily より）

上下スクロールなら下端、左右スクロールなら右端をちょうど見切れさせるってやつ。わかる〜。でもデバイスのサイズ次第でコントロールしきれない気もするんだよなぁ。

## 深掘り・補足

### 「False Bottom（偽の底）」とは

ページの下端がコンテンツの切れ目と完全に重なってしまうと、ユーザーはスクロール可能であることに気づかない可能性があります。この視覚的な錯覚が「false bottom（偽の底）」です。

### 実装の基本アプローチ

**重要なポイント**: コンテンツを意図的に「途中で切断」する

- 下端でコンテンツを途中で見切れさせる
- 次のセクションの一部を意図的に表示する
- ユーザーが「続きがある」と認識できる視覚的手がかりを残す

### 良い例

iPhone のテーブルビューのように：
- リストの行の区切りがスクリーンの下辺と重ならない
- 次の行が少し見えている状態
- ユーザーが自然とスクロールしたくなる

### 悪い例

- リストの最後の項目がちょうど画面の下端に収まっている
- セクションの区切りと画面の境界が一致している
- 視覚的に「ここが終わり」に見えてしまう

### デバイスサイズの問題について（ユーザーの懸念）

確かにデバイスサイズによって完全にコントロールしきれない部分はあります。しかし、以下のような対応策があります：

1. **動的な余白調整**: ビューポートの高さに応じてパディングを調整
2. **視覚的インジケーター**: スクロール可能であることを示すアイコンやシャドウ
3. **フェードアウト効果**: 下端にグラデーションをかけて「続きがある」ことを暗示
4. **スクロールバーの表示**: モバイルでも一時的にスクロールバーを表示

### 横スクロールの場合

上下スクロールと同じ原則が適用されます：
- 右端をちょうど見切れさせる
- 次のカードやアイテムの一部を見せる
- 横方向のカルーセルでよく使われるテクニック

### 関連する設計原則

- **Affordance（アフォーダンス）**: スクロール可能であることを示す手がかり
- **Visual Continuity（視覚的連続性）**: コンテンツの流れを途切れさせない
- **Progressive Disclosure（段階的開示）**: スクロールによって情報が徐々に明らかになる
