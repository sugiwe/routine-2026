# プロパティの選択肢でプレビューを見せる (Option Previews)

## ユーザーの第一印象（daily より）

デザインに関するツールではイメージ湧くけど、それ以外ではどうなんだろう？

## 深掘り・補足

### 基本原則

プロパティを適用する前にその結果を見ることができれば、ユーザーの試行錯誤の効率が高まります。これは「選択の不確実性を減らす」というユーザビリティの基本原則に基づいています。

### デザインツールでの典型的な実装例

- **フォント選択**: フォント名だけでなく、そのフォントで描画されたプレビューを表示
- **印刷レイアウト選択**: 縦置き・横置きなどのレイアウトを小さなサムネイルで可視化
- **筆選択**: ブラシの種類を実際のストロークで表示
- **スライドマスター選択**: テンプレートの外観を一目で確認可能

### デザインツール以外での応用例

ユーザーの疑問「それ以外ではどうなんだろう？」への回答：

#### 1. EC サイト・商品カスタマイズ
- **色選択**: 服やスマホケースの色バリエーションを画像プレビュー
- **家具配置**: AR で部屋に家具を配置したプレビュー
- **カスタムプリント**: Tシャツのデザインをリアルタイムプレビュー

#### 2. Web サービス設定
- **テーマ選択**: ダークモード/ライトモードの切り替えをライブプレビュー
- **レイアウト設定**: ダッシュボードのウィジェット配置を視覚的に確認
- **通知設定**: 通知の表示形式をサンプルで提示

#### 3. フォーム・入力インターフェース
- **日付選択**: カレンダーピッカーで選択中の日付をハイライト
- **スライダー**: 音量や明るさの変更を即座に反映
- **書式設定**: メール編集時のフォント・サイズ・色の変更をリアルタイム反映

### ユーザビリティ向上のポイント

1. **試行錯誤コストの削減**: 「適用 → 確認 → 元に戻す」のサイクルが不要に
2. **意思決定の高速化**: 視覚的な情報により直感的な選択が可能
3. **エラー防止**: 誤った選択をする前に気づける
4. **学習効果**: プレビューを見ることで選択肢の意味を理解しやすい

### 実装時の注意点

#### パフォーマンスへの配慮
- プレビュー生成に時間がかかる場合はローディング表示
- 大量の選択肢がある場合は遅延読み込み（lazy loading）
- キャッシュを活用して再描画コストを削減

#### 情報の過負荷を避ける
- すべてをプレビュー表示すると逆に混乱を招く場合も
- 重要度の高いプロパティに限定
- プレビューサイズは識別可能な最小限に

#### アクセシビリティ
- プレビューだけに頼らず、テキストラベルも併記
- 視覚障害者向けに代替テキストを提供
- キーボード操作でもプレビューを確認可能に

### 関連する UX パターン

- **即時フィードバック**: 変更を即座に反映
- **元に戻す (Undo)**: プレビュー確認後でも変更を取り消せる安心感
- **デフォルト値の提示**: 推奨設定をプレビュー付きで提案
