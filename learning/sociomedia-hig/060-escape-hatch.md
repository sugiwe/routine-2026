# 60. エスケープハッチ (Escape Hatch)

- 日付: 2026-03-01
- URL: https://www.sociomedia.co.jp/9627
- daily 記録: [daily/2026-03-01.md](../../daily/2026-03-01.md)

## ユーザーの第一印象（daily より）

いつでもホームに戻れるのは大切。iPhoneはホームボタン無くなったけど…😂

## 深掘り・補足

### エスケープハッチとは

「いつでもすぐに最初の場所に戻れるよう避難口を用意する」という原則です。ユーザーが迷子になったり、作業を中断したい場面で、基本画面への安全な帰還経路を提供します。

### 重要なポイント

1. **心理的安全性の提供**
   - ユーザーは「いつでも戻れる」という安心感があると、より積極的に探索できる
   - 不安定な状態から抜け出せる明確な手段があることで、ストレス軽減

2. **特に重要な場面**
   - 編集モードなどの特定モードでの作業時
   - 深い階層構造のナビゲーション
   - 初めて使うアプリや機能

3. **常にアクセス可能であること**
   - 出発点への復帰手段は常に見える場所にあるべき
   - 隠れたショートカットだけでなく、視覚的に明確な UI 要素として提供

### 具体例

**良い例：**
- **iPhone のホームボタン**（物理ボタン時代）: どんな状態でも押せばホーム画面に戻れる
- **Mac システム環境設定の「すべて見る」ボタン**: 各設定画面から一覧画面に即座に戻れる
- **Keynote のマスター編集モード完了ボタン**: 編集モードから通常モードへの明確な脱出口

**注意点：**
- iPhone はホームボタンが無くなり、ジェスチャー操作（画面下から上にスワイプ）に変更
- ジェスチャーは視覚的に見えないため、初心者には発見しにくい
- ただし慣れれば素早く操作できるというトレードオフ

### 実装時の考慮事項

1. **明確な視覚的表示**: 「ホームに戻る」「キャンセル」などのボタンを常に表示
2. **一貫した配置**: 同じ位置に配置することで予測可能性を高める
3. **キーボードショートカット**: Esc キーなど、物理的な脱出手段も提供
4. **確認ダイアログ**: 未保存データがある場合は確認してから戻る

### 関連する原則

- **ウェイファインディング**: 現在地と移動経路を明確にする
- **メニュー項目の位置を変化させない**: 予測可能性の向上
- **左が戻るで右が進む**: 一貫したナビゲーション方向
- **モバイルでは包括的より階層的に**: 階層が深くなるほどエスケープハッチが重要

### まとめ

エスケープハッチは、ユーザーに「安全な避難経路」を提供することで、心理的安全性を高め、より積極的な探索を促す重要な原則です。特に複雑なアプリケーションや深い階層構造を持つ UI では必須の要素と言えます。
