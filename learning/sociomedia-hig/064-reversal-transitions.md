# トランジションは両方向につける (Reversal Transitions)

- 日付: 2026-03-05
- URL: https://www.sociomedia.co.jp/9711
- daily 記録: [daily/2026-03-05.md](../../daily/2026-03-05.md)

## ユーザーの第一印象（daily より）

ページに載ってる例の2つめ、画像だとちょっと意味わからなかったけど、要は開く時と閉じるときで両方向にトランジションをつけようってことね

## 深掘り・補足

### 基本原則

画面の状態 A から状態 B への変化にトランジションのアニメーションをつける際は、その逆方向の動きを状態 B から状態 A に戻る変化にもつける、という原則です。

### なぜ重要なのか

#### 1. 情報空間の概念構造を理解しやすくする

- **対称的なアニメーション**により、ユーザーは「どこから来て、どこに戻るか」を直感的に理解できる
- 一方向だけのアニメーションでは、情報の階層関係が不明確になりがち

#### 2. 予測可能性の向上

- 開く動きと閉じる動きが対応していると、次に何が起こるかを予測しやすい
- **認知負荷の軽減**につながる

#### 3. 一貫性の維持

- UI 全体で対称的なトランジションを採用することで、システム全体の一貫性が高まる
- ユーザーの**心的モデル**が強化される

### 実践例

#### ✅ 良い例：モーダルダイアログ

- **開く**：画面下から上にスライドインしながらフェードイン
- **閉じる**：画面上から下にスライドアウトしながらフェードアウト
- → 対称的な動きで、どこに情報が戻っていくのかが明確

#### ✅ 良い例：メニューの展開/折りたたみ

- **展開**：メニューが下方向に伸びる
- **折りたたみ**：メニューが上方向に縮む
- → 視覚的に「元の状態に戻る」ことが伝わる

#### ❌ 悪い例：非対称なトランジション

- **開く**：フェードインで表示
- **閉じる**：瞬時に消える
- → ユーザーは「どこに消えたのか」が分からず、混乱する

### 関連する原則

- **一貫性の原則**：UI 全体で同じパターンを繰り返すことで、学習効果が高まる
- **フィードバックの原則**：アニメーションは操作結果の視覚的フィードバックでもある
- **空間認識**：アニメーションの方向性が情報階層を暗黙的に伝達する

### 実装のヒント

- CSS の `transition` や `animation` で、`:enter` と `:leave` の動きを対称的に設計
- フレームワーク（React, Vue など）のトランジション機能を活用
- イージング関数も対称的に設定すると、より自然な動きになる

### まとめ

トランジションを両方向につけることで、ユーザーは **「どこから来て、どこに戻るか」** を直感的に理解できます。これは単なる装飾ではなく、情報空間の構造を伝える重要なデザイン手法です。
