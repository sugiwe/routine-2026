# 多言語化を想定したUIではラベルの長さの違いを考慮する (Consider Label Length For Multilingualization)

わかりみが深い

## 深掘り・補足

### 言語による文字数の違い

多言語対応の UI を設計する際、言語によってラベルの長さが大きく変わることを考慮する必要があります。

**言語別の長さの目安（英語を基準 = 1.0 とした場合）：**

- **韓国語（ハングル）**: 約 0.75 倍（短い）
- **英語**: 基準（1.0 倍）
- **ドイツ語**: 約 1.5 倍（長い）

### 重要なポイント

#### 1. 長い言語への対応

**ドイツ語でのテスト推奨**

- ドイツ語は平均で英語の 1.4 倍の長さになる
- レイアウトの柔軟性を簡易的にテストするのに最適
- ドイツ語で問題なく表示できれば、多くの言語に対応できる可能性が高い

**対応策：**

- ボタンやナビゲーション要素に十分な余白を確保
- テキストが長くなっても折り返しや切り詰めが自然に行われるレイアウト設計
- 固定幅ではなく、可変幅のコンテナを使用

#### 2. 短い言語への対応

**韓国語や日本語（漢字）の場合**

- 文字数が少ないため、ボタンなどのタップ領域が小さくなりがち
- **押しにくくならないよう工夫が必要**

**対応策：**

- タップ領域の最小サイズを確保（通常 44×44 ピクセル以上）
- テキストが短くても、ボタンのパディングで十分なサイズを維持
- `min-width` や `min-height` を設定

### 良い例と悪い例

**❌ 悪い例 1: 固定幅のボタン（長い言語で問題）**

```
固定幅 100px のボタン
[  Save  ]  ← 英語ではOK
[Speiche…] ← ドイツ語ではテキストが切れる
[  保存  ]  ← 日本語では問題なし
```

**❌ 悪い例 2: 可変幅のボタン（最小幅なし、短い言語で問題）**

```
可変幅のボタン（最小幅なし）
[Save]  ← 英語
[Speichern]  ← ドイツ語でも余裕がある
[保存]      ← 日本語では小さすぎて押しにくい
```

**✅ 良い例: 可変幅 + 最小幅のボタン**

```
可変幅 + min-width のボタン
[  Save     ]  ← 英語
[Speichern  ]  ← ドイツ語でも余裕がある
[  保存     ]  ← 日本語でも十分なサイズが確保される
```

### 実践的なガイドライン

1. **設計段階でのテスト**
   - 複数言語（特にドイツ語など長い言語）でモックアップを作成
   - テキストが最大長になった場合のレイアウトを確認

2. **柔軟なレイアウト設計**
   - フレックスボックスやグリッドレイアウトを活用
   - 固定幅を避け、`max-width` と `min-width` を適切に設定

3. **タップ領域の確保**
   - ボタンやリンクは最小 44×44 ピクセル（iOS HIG）または 48×48 ピクセル（Material Design）を確保
   - パディングでサイズを調整

4. **フォールバック対応**
   - テキストが長すぎる場合の省略記号（`...`）の表示
   - ツールチップで完全なテキストを表示

### 関連する原則やガイドライン

- **一貫性の原則**: すべての言語で同じ UI 構造を維持
- **アクセシビリティ**: タップ領域のサイズは WCAG 2.1 の基準（最小 44×44 ピクセル）を満たす
- **レスポンシブデザイン**: 画面サイズだけでなく、テキスト長の変化にも対応
