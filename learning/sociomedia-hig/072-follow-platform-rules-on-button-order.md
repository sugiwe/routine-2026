# 肯定／否定ボタンの順序はプラットフォームのルールに従う (Follow The Platform Rules On Button Order)

- 日付: 2026-03-13
- URL: https://www.sociomedia.co.jp/9758
- daily 記録: [daily/2026-03-13.md](../../daily/2026-03-13.md)

## ユーザーの第一印象（daily より）

Windowsだけ逆やん、、、、という気持ちになった😇

## 深掘り・補足

### 基本原則：ユーザーの習慣を最優先

ダイアログにおける肯定/否定ボタンの並び順は、**アプリ内の論理よりもユーザーの習慣を優先**し、プラットフォームごとのルールに従うべきです。

### プラットフォーム別の標準

各プラットフォームには独自のガイドラインがあります：

- **macOS / iOS**: 右側が肯定ボタン（OK、保存など）
- **Windows**: 左側が肯定ボタン（はい、OKなど）
- **Android**: 右側が肯定ボタン

ユーザーがおっしゃる通り、**Windows だけが他のプラットフォームと逆**になっています。これは歴史的な経緯によるもので、Windows のガイドラインでは「タブオーダー（Tab キーで移動する順序）」を重視し、左から右へ進む際に肯定ボタンを先に配置する設計思想があります。

### なぜプラットフォームのルールに従うべきか

1. **ユーザーの学習コストを最小化**
   - ユーザーは OS 全体で一貫した操作体系に慣れている
   - 独自ルールを採用すると、毎回考えて操作する必要が生じる

2. **筋肉記憶への配慮**
   - 「右側をクリックすれば進める」といった無意識の操作パターンを尊重
   - プラットフォーム内で一貫性があれば、ミスクリックが減る

3. **アクセシビリティ**
   - スクリーンリーダーやキーボード操作での予測可能性が向上

### 一般的な感覚：左が戻る、右が進む

プラットフォーム固有のルールがない場合は、**「左が戻るで右が進む」**という直感的な操作感を基準にします。これは：

- 横書き文化圏（LTR: Left-to-Right）における自然な流れ
- ウィザードやページネーションでの標準的な配置

### RTL 言語への配慮

アラビア語やヘブライ語など、**右から左へ書く言語（RTL: Right-to-Left）**では、この原則が逆転します：

- **右が戻る、左が進む**が自然
- UI 全体を鏡像反転させる必要がある

### 実装時のポイント

- プラットフォームの公式ガイドラインを確認
- ネイティブ UI コンポーネントを使えば自動的に標準に従える
- クロスプラットフォームアプリでは、各 OS ごとにボタン順序を調整する

### まとめ

「Windows だけ逆」という感覚は正しく、これはまさにプラットフォームごとの慣習の違いを示しています。重要なのは、**各プラットフォームのユーザーが慣れ親しんだ配置に従うこと**です。独自の美学や論理よりも、ユーザーの習慣を優先することが、優れた UX につながります。
