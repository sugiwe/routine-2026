## 深掘り・補足

### プロスペクティブメモリーとは

**プロスペクティブメモリー**とは、「未来の自分のために手がかりを残せるようにする」という UI デザイン原則です。ユーザーが後から参照したい情報やコンテンツへのアクセスを容易にすることを目的としています。

### なぜ重要なのか

人間の記憶には限界があり、特に「後でやろうと思っていたこと」を忘れてしまうことは日常的です。プロスペクティブメモリーの仕組みを提供することで：

- **認知負荷を軽減**: ユーザーが「覚えておく」必要性を減らす
- **作業の中断と再開**: 一度離れても、簡単に元の場所に戻れる
- **情報の整理**: 後で見返すべき情報を明示的にマークできる

この原則は「ユーザーの記憶に頼らない」という UI デザインの基本哲学に直結しています。

### 具体的な実装例

プロスペクティブメモリーを支援する機能には、以下のようなものがあります：

#### 1. ブックマーク機能
- Web ブラウザの「お気に入り」や「ブックマーク」
- 後で読みたいページを保存

#### 2. フラグ付け・いいね機能
- メールアプリの「フラグ」
- SNS の「いいね」や「保存」ボタン
- 重要なアイテムや気になったコンテンツにマーク

**ユーザーの疑問への回答**: はい、いいねボタンもプロスペクティブメモリーの一種と言えます！単なる「評価」だけでなく、「後で見返したいコンテンツ」を記録する手段としても機能します。

#### 3. リーディングリスト
- iPhone Safari の「リーディングリストに追加」
- 「後で読む」専用の保存場所

#### 4. 下書き保存機能
- メール・投稿の下書き保存
- 作業中断後、簡単に再開できる

#### 5. ウィンドウやタブの保持
- ブラウザのセッション復元
- アプリの状態保存

#### 6. 仮想付箋やマーカー
- PDF リーダーのハイライト機能
- ノートアプリの付箋機能

### 設計のポイント

#### ✅ 良い設計

1. **複数の保存方法を提供**
   - 例: Safari の「ブックマーク」「リーディングリスト」「ホーム画面に追加」など
   - ユーザーの目的に応じて使い分けられる

2. **アクセスしやすい場所に配置**
   - 保存した情報に素早くアクセスできる UI

3. **視覚的なフィードバック**
   - 保存済みかどうかが一目でわかる（塗りつぶされたアイコンなど）

4. **整理・分類のサポート**
   - フォルダやタグで分類できる
   - 検索機能で見つけやすい

#### ❌ 避けるべき設計

1. **保存方法が分かりにくい**
   - 機能が隠れていたり、アイコンの意味が不明確

2. **保存した情報にアクセスできない**
   - 保存はできるが、どこにあるか分からない
   - 保存リストが整理できず、探すのが困難

3. **一時的な情報と永続的な情報が混在**
   - 「最近見たもの」と「明示的に保存したもの」が区別できない

### 関連する原則

- **ユーザーの記憶に頼らない**: システムが情報を保持し、ユーザーの認知負荷を減らす
- **可視性 (Visibility)**: 保存した情報や機能が見えやすい状態にする
- **フィードバック**: 保存したことが明確に分かるフィードバックを提供

### まとめ

プロスペクティブメモリーは、「未来の自分」をサポートする設計原則です。ユーザーが「後でやろう」「後で見よう」と思ったときに、その意図を簡単に記録し、後から確実に実行できるようにする仕組みを提供することが重要です。
