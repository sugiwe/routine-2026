# 80. ドリルダウンは上→下、左→右 (Top-Bottom, Left-Right Drilldown Orientation)

- 日付: 2026-03-21
- URL: https://www.sociomedia.co.jp/9796
- daily 記録: [daily/2026-03-21.md](../../daily/2026-03-21.md)

## ユーザーの第一印象（daily より）

へー、スマホのフッターナビゲーションはこの原則からすると「❌」なんだ👀「許容されている」と書かれているけど、自分はもうちょっと積極的に「望ましい」という感じで捉えている。

## 深掘り・補足

### 基本原則

階層化されたナビゲーション設計において、以下の位置関係を保つことが推奨されます：

- **上で選択した項目の内容が下に表示される**
- **左で選択した項目の内容が右に表示される**

この原則は、直感的な情報構造を実現し、ユーザーが階層間の関係を理解しやすくするデザイン手法です。

### RTL言語への対応

アラビア語など右から左に読む言語（RTL: Right-to-Left）では、左右関係を逆にする必要があります。文化的・言語的な違いに適応させることが重要です。

### スマートフォンでの例外

スマートフォンアプリでは、ナビゲーションの最上位項目をスクリーン下部に配置するデザインがよく見られます（フッターナビゲーション）。

この場合、技術的には「上→下」の原則に反しますが、**狭いスクリーンにおいてスクリーン上部の標準的なヘッダーと共存させる表現として、許容されている**とされています。

#### ユーザーの視点

ユーザーさんが指摘されているように、「許容されている」という表現は消極的に聞こえるかもしれません。実際には、以下の理由からフッターナビゲーションは**望ましい設計**と言えます：

- **親指での操作性**: スマートフォンを片手で持った際、画面下部は親指で最も届きやすいエリア
- **認知負荷の軽減**: 常に同じ位置にナビゲーションがあることで、迷わずアクセスできる
- **視線移動の効率**: コンテンツを見た後、自然な流れで下部のナビゲーションに視線が移動する

### 良い例と悪い例

#### 良い例
- デスクトップアプリで左側にフォルダツリー、右側にフォルダ内容を表示
- Webサイトで上部にカテゴリ、その下に商品一覧を表示
- スマホアプリで画面下部にタブナビゲーション

#### 悪い例
- 左側に詳細コンテンツ、右側にカテゴリ選択（階層関係が逆）
- 下部にカテゴリ、上部に詳細（情報の流れが不自然）

### 関連する原則

- **空間記憶（Spatial Memory）**: 一貫した配置により、ユーザーは位置を記憶しやすくなる
- **自然なマッピング（Natural Mapping）**: 物理的な配置が概念的な階層と一致することで理解しやすくなる
