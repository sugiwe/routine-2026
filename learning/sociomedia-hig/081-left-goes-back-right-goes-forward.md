# 81. 左が戻るで右が進む (Left Goes Back, Right Goes Forward)

- 日付: 2026-03-22
- URL: https://www.sociomedia.co.jp/9798
- daily 記録: [daily/2026-03-22.md](../../daily/2026-03-22.md)

## ユーザーの第一印象（daily より）

言語で左右反転することがあるのは面白いし、気をつけないとだ

## 深掘り・補足

### 原則の定義

画面遷移の方向性を水平軸で表現する際のメンタルモデルを確立する原則です。基本的には「**左を戻る（過去）に、右を進む（未来）に対応させる**」という考え方です。

### 重要なポイント

#### 1. 言語による左右反転

最も注目すべき点は、**この原則が普遍的ではない**ということです。

- **LTR言語（英語、日本語など）**: 左が戻る、右が進む
- **RTL言語（アラビア語、ヘブライ語など）**: **左右関係をすべて逆転させる必要がある**

これは文化的・言語的なユーザーの認知パターンを尊重する設計思想を示しています。

#### 2. ユーザーの直感に合わせる

この原則は、ユーザーが日常的に使っている言語の読み書き方向と一致させることで、直感的な操作を実現します。

### 具体例

#### 良い例：LTR言語での実装

```
[← 戻る]  [進む →]
```

- カレンダーアプリの月送り
- スライドショーのナビゲーション
- ブラウザの「戻る」「進む」ボタン

#### 良い例：RTL言語での実装

```
[← לְהִמשִׁיךְ]  [לַחֲזוֹר →]
```

（ヘブライ語の場合、左が「進む」、右が「戻る」）

### 注意点

1. **ターゲットユーザーの言語体系を事前に確認する**
   - グローバル展開するアプリでは、言語設定に応じて動的に切り替える必要がある

2. **単なる反転では不十分**
   - アイコンの向きも逆転させる
   - レイアウト全体の左右を反転させる

3. **テストの重要性**
   - RTL環境でのテストを必ず実施する

### 関連する原則

- **ドリルダウンは上→下、左→右**: 階層構造のナビゲーションにおける方向性の原則
- **空間記憶 (Spatial Memory)**: ユーザーが要素の位置を記憶することを活用する原則

### 実装のベストプラクティス

1. **CSS の `dir` 属性や `direction` プロパティを活用**
   ```css
   [dir="rtl"] .navigation {
     flex-direction: row-reverse;
   }
   ```

2. **アイコンは論理的な方向を使う**
   - `chevron-start` / `chevron-end` のような論理名を使う
   - `chevron-left` / `chevron-right` のような物理名は避ける
