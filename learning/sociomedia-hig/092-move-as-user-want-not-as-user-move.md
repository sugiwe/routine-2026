# 92. 動かしたままに動くのではなく動かしたいように動く (Move As User Want, Not As User Move)

- 日付: 2026-04-02
- URL: https://www.sociomedia.co.jp/9876
- daily 記録: [daily/2026-04-02.md](../../daily/2026-04-02.md)

## ユーザーの第一印象（daily より）

階層式のメニューの例、確かにそういうことあるかも👀

## 深掘り・補足

### 原則の本質

このガイドラインは、インターフェースがユーザーの入力に**忠実に応答するだけでは不十分**であることを示しています。重要なのは、ユーザーが「思いどおりにコントロールできている」と感じられることです。

単なる入力値の直接反映ではなく、**「あそび」と「補正」を組み込む**ことが鍵となります。これにより、ユーザーは入力デバイスに支配されるのではなく、自らの意図を正確に反映させられていると感じられます。

### 具体的な実装例

#### 1. 階層式メニュー

**問題**: 階層式のドロップダウンメニューで、親項目から子項目にマウスを移動する際、斜めに動かすとすぐに子階層が閉じてしまう。

**解決策**: 親項目上でマウスが斜めに移動する際、一定時間の猶予を設ける。これにより、ユーザーが子項目に到達する前にメニューが閉じるのを防ぐ。

**効果**: ユーザーは「メニューが自分の意図を理解してくれている」と感じる。

#### 2. マウスポインターのアクセラレーション

**仕組み**: マウスの移動速度に応じて画面上のポインターの移動量を調整。
- **ゆっくりした動き**: 小幅に反映（精密な操作が可能）
- **速い動き**: 大幅に反映（画面全体を素早く移動）

**効果**: ユーザーは物理的なマウスの移動量に縛られず、意図した位置に素早く正確に到達できる。

#### 3. iPhone のタッチ操作補正

**仕組み**: 画面上部ほど上方向への補正を加える。これは、ユーザーが画面を下からやや見上げた姿勢で操作していることを前提とした補正。

**効果**: ユーザーが「タップしたい」と思った位置と、実際にタップした位置のズレを自動的に修正。自然な操作感を実現。

### ユーザー体験への影響

これらの補正により、ユーザーは以下のように感じます：

- ✅ **「自分が思った通りに動いてくれる」**: 入力デバイスの物理的制約から解放される
- ✅ **「コントロールできている」**: システムが自分の意図を理解してくれていると感じる
- ✅ **「操作がスムーズ」**: ストレスなく目的の操作を実行できる

### 実践的なポイント

1. **物理的な入力をそのまま反映しない**: ユーザーの意図を推測し、適切な補正を加える
2. **「あそび」を設ける**: 誤操作を防ぐため、一定の猶予や閾値を設定
3. **文脈を考慮**: デバイスの特性や使用状況に応じて補正を調整
4. **過度な補正は避ける**: ユーザーが「勝手に動かされている」と感じないように注意

### 悪い例

- ❌ マウスの動きをピクセル単位で厳密に反映し、階層メニューがすぐに閉じてしまう
- ❌ タッチ位置を補正せず、ユーザーが何度もタップし直す必要がある
- ❌ スクロールの慣性が全くなく、指を離した瞬間に停止してしまう

### 良い例

- ✅ 階層メニューに遅延を設け、ユーザーが子項目に到達しやすくする
- ✅ タッチ操作に補正を加え、ユーザーの意図した位置を正確に認識
- ✅ スクロールに慣性をつけ、自然な「流れ」を演出
