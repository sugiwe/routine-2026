# 文字の拡大に対応する (Support Text Enlargement)

## ユーザーの第一印象（daily より）

重要だけどしっかり対応するの大変なやつだ

## 深掘り・補足

### 原則の概要

このガイドラインの核心は、**「文字の表示サイズを大きくして利用できるよう、プラットフォームやブラウザのテキスト拡大機能に対応する」** ことです。ユーザーが自身の視力や好みに合わせて文字サイズを調整できる環境の構築が目的です。

### なぜ重要か

- **視覚障害者のアクセシビリティ向上**: 弱視のユーザーや高齢者にとって、文字を拡大できることは UI を使えるかどうかの分かれ目になる
- **法的要求**: WCAG（Web Content Accessibility Guidelines）では文字の拡大対応が求められている
- **多様なユーザー層への対応**: 視力の状況は人それぞれ異なり、柔軟な対応が求められる

### 技術的な実装方法

#### ✅ 推奨されるアプローチ

1. **相対単位の使用**
   - `em`, `rem`, `%` などの相対単位を使う
   - 固定ピクセルサイズ（`px`）は避ける
   - ユーザーのブラウザ設定に応じて柔軟に拡大・縮小できる

2. **標準仕様への準拠**
   - OS・ブラウザネイティブのズーム機能との互換性を確保
   - プラットフォーム標準の文字表示の実装を使う

3. **レスポンシブデザインの応用**
   - レイアウトが崩れないように、文字拡大に対応したデザイン
   - コンテナの幅や高さも柔軟に調整されるように

#### ❌ 避けるべき実装

- **固定ピクセルサイズの使用**: `font-size: 14px` のような固定値は拡大に対応できない
- **`user-scalable=no` の使用**: モバイルでユーザーのピンチズーム操作を無効化してはいけない
- **オーバーフローの無視**: 文字が拡大されたときにコンテンツが隠れたり、レイアウトが崩れたりする

### 関連するガイドライン

- **94. 音声読み上げに対応する**: アクセシビリティ対応の一環として、視覚障害者への配慮
- **96. 色に依存させない**: 視覚的な情報伝達において、複数の方法でユーザーに情報を提供する

### 実装時の注意点

- **テストの重要性**: ブラウザの拡大機能（200%、300% など）でレイアウトが崩れないかテストする
- **最小文字サイズの設定**: あまりにも小さい文字サイズをデフォルトにしない
- **コンテンツの優先順位**: 文字が拡大されたときに重要な情報が隠れないように、コンテンツの優先順位を明確にする

### まとめ

文字の拡大対応は「重要だけどしっかり対応するの大変なやつ」というユーザーの第一印象通り、技術的には地味だけど非常に重要なアクセシビリティ対応です。標準仕様に準拠した実装と相対単位の使用で、多様なユーザーが快適に UI を利用できる環境を整えましょう。
